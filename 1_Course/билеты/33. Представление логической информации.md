### **33. Представление логической информации**

**Логическая (булева) информация** — данные, принимающие два значения: **«истина»** и **«ложь»**.

#### **Физическое представление**
- В ЭВМ:  
  - **0** → ложь,  
  - **любое ненулевое значение** → истина.  
- На аппаратном уровне — уровни напряжения (низкий/высокий).

#### **На уровне процессора**
- Отдельного типа «булево» **нет** — используются **целые числа**.
- Результат логических операций фиксируется в **флагах регистра состояния (PS)**:
  - **Z (Zero)** = 1 ↔ результат = 0 ↔ ложь,
  - **Z = 0** ↔ результат ≠ 0 ↔ истина.

#### **Логические операции**
Реализуются как **побитовые команды**:
- `NOT` — побитовое отрицание,
- `AND`, `OR`, `XOR` — побитовые И, ИЛИ, исключающее ИЛИ.

#### **Условные переходы**
Используют флаги для управления потоком:
- `BEQ` / `BNE` — если Z = 1 / Z = 0,
- `BLT` / `BGE` — знаковое сравнение (N, V),
- `BCC` / `BCS` — беззнаковое сравнение (C).

# 33. Представление логической информации

### **1. Что такое логическая информация?**

**Логическая (булева) информация** — это данные, принимающие **только два значения**:  
- **«истина» (true)**  
- **«ложь» (false)**

Эти значения лежат в основе **логических операций** (И, ИЛИ, НЕ), **условных конструкций** и **управления потоком выполнения** программ.

---

### **2. Физическое представление в ЭВМ**

В цифровых системах логические значения кодируются **электрическими уровнями**:
- **0 В (низкий уровень)** → **логический 0** → **«ложь»**,
- **+5 В / +3.3 В / +1.8 В (высокий уровень)** → **логическая 1** → **«истина»**.

На уровне машинного слова логическое значение может быть представлено:
- **одним битом** (наиболее эффективно),
- **целым байтом или словом** (для удобства адресации).

> 💡 В большинстве архитектур:  
> - **0** = ложь,  
> - **любое ненулевое значение** = истина.

---

### **3. Представление в языках программирования**

| Язык | Ложь | Истина |
|------|------|--------|
| C | `0` | Любое ≠ 0 |
| C++ / Java / Python | `false` / `0` / `None` / пустая строка | `true` / ≠0 / непустой объект |
| Ассемблер | `0` | ≠ `0` |

> ⚠️ В ассемблере и на уровне процессора **нет отдельного типа «булево»** — используются **целые числа**, где:
> - **ZF (Zero Flag) = 1** ↔ результат = 0 ↔ **ложь**,
> - **ZF = 0** ↔ результат ≠ 0 ↔ **истина**.

---

### **4. Логические операции на уровне процессора**

Процессоры реализуют базовые логические операции как **побитовые команды**:

| Операция | Обозначение | Пример (8 бит) |
|---------|------------|----------------|
| **НЕ** | `NOT` | `NOT 01010101 = 10101010` |
| **И** | `AND` | `0101 AND 0011 = 0001` |
| **ИЛИ** | `OR` | `0101 OR 0011 = 0111` |
| **ИСКЛЮЧАЮЩЕЕ ИЛИ** | `XOR` | `0101 XOR 0011 = 0110` |

Эти операции применяются:
- Для **маскирования битов**,
- Для **установки/сброса флагов**,
- В **криптографии и хешировании**.

---

### **5. Условные переходы и флаги**

Результат логического сравнения сохраняется в **флагах регистра состояния (PS)**:

| Флаг | Значение | Используется в условных переходах |
|------|---------|----------------------------------|
| **Z (Zero)** | 1, если результат = 0 | `BEQ` (Branch if Equal), `BNE` |
| **N (Negative)** | 1, если результат < 0 | `BLT`, `BGE` |
| **C (Carry)** | Перенос из старшего бита | `BCC`, `BCS` |
| **V (Overflow)** | Переполнение знакового результата | `BVS`, `BVC` |

> 💡 Например, команда `CMP A, B` вычитает `B` из `A` и устанавливает флаги, но **не сохраняет результат**.  
> Затем `BEQ` переходит, если `A == B` (т.е. Z = 1).

---

### **6. Представление в БЭВМ-NG**

В методических указаниях по БЭВМ-NG **отдельно не выделено понятие «логическая информация»**, но оно **реализовано через целочисленную арифметику и флаги**:

- Команды `CMP`, `SUB` устанавливают флаги **Z, N, V, C**,
- Команды ветвления (`BEQ`, `BNE`, `BPL`, `BMI` и др.) интерпретируют эти флаги как **логические условия**,
- **Нет отдельного типа «булево»** — всё сводится к **целым числам и флагам**.

> 📌 Пример:  
> ```asm
> CMP AX, 0   ; Сравнить AX с нулём
> BEQ metka   ; Перейти, если AX == 0 (логическое "ложь")
> ```

---

### **7. Заключение**

Логическая информация в ЭВМ — это **фундаментальный концепт**, реализуемый через:
- **бинарное представление (0/1)**,
- **побитовые логические операции**,
- **систему флагов и условных переходов**.

Хотя на аппаратном уровне нет отдельного «булевого типа», вся логика программ (ветвления, циклы, проверки) строится на **интерпретации целочисленных результатов как логических значений**. Это делает представление логической информации **универсальным, эффективным и глубоко интегрированным** в архитектуру процессора.
