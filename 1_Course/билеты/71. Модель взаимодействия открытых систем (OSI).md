# 71. Модель взаимодействия открытых систем (OSI)

### **1. История и назначение**

**Модель OSI (Open Systems Interconnection)** была разработана **Международной организацией по стандартизации (ISO)** в **1984 году** с целью:
- обеспечить **унификацию сетевых протоколов**,
- упростить **проектирование и отладку** сетевых систем,
- обеспечить **совместимость оборудования** разных производителей.

Хотя на практике чаще используется **модель TCP/IP**, модель OSI остаётся **фундаментальной теоретической основой** для понимания сетевых технологий.

---

### **2. Общая структура: 7 уровней**

Модель OSI разделяет сетевое взаимодействие на **семь логических уровней**, каждый из которых выполняет **чётко определённую функцию**. Уровни нумеруются **снизу вверх**:

| № | Название уровня | Основная функция |
|---|------------------|------------------|
| **7** | **Прикладной (Application)** | Интерфейс для приложений |
| **6** | **Представительский (Presentation)** | Преобразование данных |
| **5** | **Сеансовый (Session)** | Управление сеансами связи |
| **4** | **Транспортный (Transport)** | Надёжная доставка данных |
| **3** | **Сетевой (Network)** | Маршрутизация и логическая адресация |
| **2** | **Канальный (Data Link)** | Доставка по физическому каналу |
| **1** | **Физический (Physical)** | Передача битов по среде |

> 💡 **Мнемоника**: **П**апа **П**ил **С**амогон **Т**ринадцатого **С**ентября **К**ак **Ф**ранцуз → **П**рикладной, **П**редставительский, **С**еансовый, **Т**ранспортный, **С**етевой, **К**анальный, **Ф**изический.

### **3. Подробное описание каждого уровня**

#### **Уровень 7: Прикладной (Application)**
- **Функция**: предоставляет сетевые сервисы конечным приложениям.
- **Протоколы**: HTTP, FTP, SMTP, DNS, SSH, Telnet.
- **Пример**: браузер запрашивает веб-страницу через HTTP.

#### **Уровень 6: Представительский (Presentation)**
- **Функция**: преобразование данных в формат, понятный приложению.
- **Задачи**:
  - Кодирование/декодирование (MIME, ASCII/UTF-8),
  - Шифрование/дешифрование (SSL/TLS на этом уровне),
  - Сжатие данных.
- **Пример**: преобразование JPEG в RGB-пиксели.

#### **Уровень 5: Сеансовый (Session)**
- **Функция**: управление диалогом между приложениями.
- **Задачи**:
  - Установление, поддержание и завершение сеанса,
  - Синхронизация (точки восстановления),
  - Управление полным/половинным дуплексом.
- **Пример**: сеанс видеочата, синхронизация баз данных.

#### **Уровень 4: Транспортный (Transport)**
- **Функция**: **надёжная сквозная доставка** данных между процессами.
- **Основные протоколы**:
  - **TCP** — с установлением соединения, контролем ошибок, повторной передачей.
  - **UDP** — без установления соединения, быстрый, но ненадёжный.
- **Механизмы**:
  - Нумерация сегментов,
  - Подтверждения (ACK),
  - Управление потоком (окна),
  - Управление перегрузками.
- **Адресация**: **порты** (0–65535).

#### **Уровень 3: Сетевой (Network)**
- **Функция**: **маршрутизация** пакетов между сетями.
- **Протоколы**: IP (IPv4, IPv6), ICMP, ARP, OSPF, BGP.
- **Адресация**: **IP-адреса** (логические, маршрутизируемые).
- **Задачи**:
  - Фрагментация и сборка пакетов,
  - Выбор маршрута,
  - Обработка ошибок (через ICMP).

#### **Уровень 2: Канальный (Data Link)**
- **Функция**: **доставка кадров** по физическому каналу **в пределах одной сети**.
- **Подуровни**:
  - **LLC (Logical Link Control)** — управление логическим соединением.
  - **MAC (Media Access Control)** — контроль доступа к среде.
- **Протоколы**: Ethernet (IEEE 802.3), Wi-Fi (802.11), PPP.
- **Адресация**: **MAC-адреса** (физические, 48 бит).
- **Механизмы**:
  - Обнаружение ошибок (CRC),
  - Управление доступом (CSMA/CD в старом Ethernet).

#### **Уровень 1: Физический (Physical)**
- **Функция**: **передача необработанных битов** по физической среде.
- **Параметры**:
  - Тип среды: витая пара, оптика, радиоканал,
  - Кодирование сигналов: Manchester, NRZ,
  - Напряжение, частота, скорость передачи.
- **Устройства**: кабели, repeater (репитер), хабы.

---

### **4. Инкапсуляция и передача данных**

При отправке данных происходит **процесс инкапсуляции**:

1. **Прикладной уровень**: создаёт **сообщение**.
2. **Представительский**: добавляет данные для шифрования/сжатия.
3. **Сеансовый**: добавляет управляющую информацию сеанса.
4. **Транспортный**: разбивает на **сегменты**, добавляет **порты** → **сегмент**.
5. **Сетевой**: добавляет **IP-заголовок** → **пакет**.
6. **Канальный**: добавляет **MAC-заголовок и CRC** → **кадр**.
7. **Физический**: преобразует кадр в **битовый поток**.

На приёмной стороне — **обратный процесс (деинкапсуляция)**.

---

### **5. Устройства и уровни OSI**

| Устройство             | Работает на уровне |
| ---------------------- | ------------------ |
| Хаб (Hub)              | 1 (Физический)     |
| Коммутатор (Switch)    | 2 (Канальный)      |
| Маршрутизатор (Router) | 3 (Сетевой)        |
| Брандмауэр (Firewall)  | 3–4 (часто выше)   |
| Шлюз (Gateway)         | 4–7                |

---

### **6. Сравнение с моделью TCP/IP**

| OSI                                        | TCP/IP              | Комментарий                     |
| ------------------------------------------ | ------------------- | ------------------------------- |
| Прикладной + Представительский + Сеансовый | Прикладной          | В TCP/IP эти функции объединены |
| Транспортный                               | Транспортный        | Аналогичен                      |
| Сетевой                                    | Интернет (Internet) | IPv4/IPv6                       |
| Канальный + Физический                     | Сетевой интерфейс   | Объединены                      |

> 💡 **TCP/IP — практичная модель**, OSI — **теоретическая**.

---

### **7. Заключение**

Модель OSI — это **иерархическая абстракция**, позволяющая:
- чётко разделять функции сетевой коммуникации,
- разрабатывать протоколы независимо на каждом уровне,
- упрощать обучение и диагностику сетей.

Несмотря на то, что реальные протоколы (например, HTTP/IP/Ethernet) не всегда строго соответствуют всем семи уровням, модель OSI остаётся **основой сетевой инженерии**.

> 📌 **Примечание**: В методических указаниях по БЭВМ-NG **модель OSI не упоминается**, так как БЭВМ — учебная модель процессора без сетевых функций.
