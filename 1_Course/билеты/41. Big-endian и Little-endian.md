### **41. Big-endian и Little-endian**

**Порядок байтов (endianness)** — соглашение о порядке хранения байтов многобайтового числа в памяти.

#### **Типы**
- **Big-endian (BE)**: старший байт по **меньшему адресу**.
- **Little-endian (LE)**: младший байт по **меньшему адресу**.

#### **Пример (32-битное число `0x12345678`)**
| Адрес | BE | LE |
|-------|----|----|
| 0x1000 | 12 | 78 |
| 0x1001 | 34 | 56 |
| 0x1002 | 56 | 34 |
| 0x1003 | 78 | 12 |

#### **Распространение**
- **LE**: x86/x86-64, ARM (по умолчанию) — ПК, смартфоны.
- **BE**: SPARC, старые PowerPC, сетевые протоколы.
- **Сетевой порядок байтов** — всегда **Big-endian**.

#### **Преобразование в сетях**
- На LE-машинах перед отправкой:
  - `htons()`, `htonl()` — host → network,
  - `ntohs()`, `ntohl()` — network → host.

#### **Влияние на программирование**
- Критично при работе с:
  - сырыми байтами (драйверы, парсеры),
  - сетевыми пакетами,
  - бинарными файлами.
- В высокоуровневом коде — скрыто ОС и компилятором.


# 41. Big-endian и Little-endian

### **1. Определение**

**Порядок байтов (endianness)** — это соглашение о том, **в каком порядке хранятся байты многобайтового числа в памяти**.

Существует два основных варианта:

| Тип | Название | Порядок |
|-----|---------|--------|
| **Big-endian** | «Старший байт первым» | Старший (наиболее значимый) байт по **меньшему адресу** |
| **Little-endian** | «Младший байт первым» | Младший (наименее значимый) байт по **меньшему адресу** |

> 💡 Названия происходят из сатирического романа Джонатана Свифта «Путешествия Гулливера», где жители Лилипутии спорили, с какой стороны разбивать яйцо — с острого (little-end) или тупого (big-end) конца.

---

### **2. Пример: число 0x12345678 в 32-битной памяти**

Предположим, число записывается по адресу `0x1000`.

| Адрес | Big-endian | Little-endian |
|-------|------------|---------------|
| 0x1000 | **12** | **78** |
| 0x1001 | **34** | **56** |
| 0x1002 | **56** | **34** |
| 0x1003 | **78** | **12** |

- **Big-endian**: читается так же, как пишется в шестнадцатеричной записи → удобно для человека.
- **Little-endian**: младший байт первым → удобно для процессора при арифметике.

---

### **3. Где используется каждый порядок?**

| Архитектура / Протокол | Порядок | Комментарий |
|------------------------|--------|------------|
| **x86, x86-64** | Little-endian | Все процессоры Intel и AMD |
| **ARM** | Настраиваемый, но по умолчанию Little-endian | В мобильных устройствах — LE |
| **PowerPC** | Настраиваемый | Использовался в старых Mac (BE), серверах (LE/BE) |
| **SPARC** | Big-endian | Серверные системы Sun |
| **TCP/IP, IPv4, IPv6** | **Big-endian** | Сетевой порядок байтов (network byte order) |
| **Файловые форматы** | Зависит от формата | Например, JPEG, PNG — BE; BMP — LE |

> ⚠️ При передаче данных между системами с разным endianness требуется **преобразование** (byte swapping).

---

### **4. Сетевой порядок байтов**

Все сетевые протоколы (TCP, UDP, IP) используют **Big-endian** — это называется **сетевым порядком байтов (network byte order)**.

- На **Little-endian машине** (например, x86) перед отправкой 16- или 32-битного числа в сеть его нужно преобразовать:
  - `htons()` — host to network short (16 бит),
  - `htonl()` — host to network long (32 бит).
- При получении — обратное преобразование:
  - `ntohs()`, `ntohl()`.

> 💡 Это гарантирует, что IP-адрес `192.168.1.1` будет корректно прочитан на любой машине.

---

### **5. Как определить endianness системы?**

Простой способ на C:

```c
#include <stdio.h>

int main() {
    unsigned int x = 0x12345678;
    unsigned char *p = (unsigned char*)&x;
    if (*p == 0x78)
        printf("Little-endian\n");
    else if (*p == 0x12)
        printf("Big-endian\n");
    return 0;
}
```

> 🔍 Если младший байт (`0x78`) лежит по нулевому смещению — система **Little-endian**.

---

### **6. Влияние на программирование**

- **Низкоуровневое программирование**: при работе с сырыми байтами (драйверы, сетевые пакеты, файловые форматы) необходимо учитывать endianness.
- **Высокоуровневое программирование**: компилятор и ОС скрывают различия — целые числа работают одинаково.
- **Кроссплатформенная разработка**: бинарные данные, сериализованные на одной машине, могут быть неверно прочитаны на другой без преобразования.

---

### **7. Endianness в БЭВМ-NG**

В методических указаниях по БЭВМ-NG **порядок байтов не обсуждается**, так как:
- Все операции выполняются над **16-битными словами целиком**,
- Нет команд для работы с отдельными байтами (например, `MOV AL, [addr]`),
- Нет сетевого взаимодействия или внешних двоичных форматов.

Тем не менее, если бы БЭВМ работала с байтами, её поведение зависело бы от принятого соглашения — но в рамках модели это **не требуется**.

---

### **8. Заключение**

Big-endian и Little-endian — это **фундаментальные соглашения**, влияющие на совместимость данных между системами.  
- **Big-endian** доминирует в **сетях и некоторых RISC-архитектурах**,  
- **Little-endian** — в **персональных компьютерах и мобильных устройствах**.

Понимание endianness критически важно при разработке сетевых приложений, драйверов, парсеров файлов и кроссплатформенного ПО. В то же время, в большинстве прикладных задач эти детали скрыты абстракциями ОС и языков программирования.

> 📚 **Источник**: общая теория архитектуры ЭВМ; RFC 1700 (сетевой порядок байтов).
