### **39. Символы: WIN1251**

**Windows-1251** — 8-битная кодовая страница от Microsoft для кириллицы, ставшая де-факто стандартом в странах СНГ (1990–2010-е гг.).

#### **Структура**
- **0–127**: совпадает с **ASCII**.
- **128–159**: **типографские символы** («», «—», «…», ™, № и др.).
- **160–255**: кириллические буквы.

#### **Расположение кириллицы**
- **Заглавные**: 192–223 (`А` = 192, `Я` = 223),
- **Строчные**: 224–255 (`а` = 224, `я` = 255),
- **Ё/ё**: 168 / 184 (вне основного блока).

#### **Преимущества**
- Наличие **полезных знаков препинания и типографики**:
  - `«` = 171, `»` = 187,
  - `—` = 151,
  - `№` = 185,
  - `…` = 133.
- Удобна для **офисной работы и веба**.

#### **Недостатки**
- Не является международным стандартом (частная кодировка Microsoft),
- При обрезке старшего бита → **абракадабра** (в отличие от КОИ-8),
- Поддерживает только **латиницу и кириллицу**.

#### **Сравнение**
| Кодировка | Типографика | Распространение |
|-----------|-------------|------------------|
| **WIN1251** | ✅ Полная | Очень высокое (Windows, веб) |
| **КОИ-8**   | ❌ Минимум | Высокое (UNIX) |
| **ISO-8859-5** | ❌ Нет | Низкое |

#### **В БЭВМ-NG**
- Символы хранятся в **младших 8 битах 16-битного слова**.  
  Пример: `'А' = 0x00C0`, `'—' = 0x0097`.

#### **Современное состояние**
- Фактически вытеснена **UTF-8**, но встречается в легаси-системах и архивах.
# 39. Символы: WIN1251

### **1. Общее определение**

**Windows-1251** (сокращённо **WIN1251** или **CP1251**) — это **8-битная кодовая страница**, разработанная компанией **Microsoft** для представления **кириллических символов** в операционных системах **Windows**.

Она была введена в **Windows 2.0 (1987 г.)** и стала **де-факто стандартом** для русскоязычного сегмента Интернета, офисных документов и программного обеспечения в странах СНГ в **1990–2010-х годах**.

---

### **2. Структура кодировки**

- **Размер**: 8 бит → **256 символов** (0–255).
- **Диапазоны**:
  - **0–127**: **полностью совпадает с ASCII** (латиница, цифры, знаки).
  - **128–159**: **дополнительные печатаемые символы** (отсутствуют в ISO-8859-5!).
  - **160–255**: **кириллические символы**.

> 💡 Ключевое преимущество WIN1251 перед ISO-8859-5 — **наличие полезных типографских символов** в диапазоне 128–159.

---

### **3. Расположение кириллических символов**

- **Заглавные буквы**: с **192 (0xC0)** по **223 (0xDF)**
- **Строчные буквы**: с **224 (0xE0)** по **255 (0xFF)**

Порядок — **почти алфавитный**, с небольшими исключениями:

| Позиция | Символ |
|--------|--------|
| 192 | А |
| 193 | Б |
| ... | ... |
| 202 | Й |
| 203 | К |
| ... | ... |
| 222 | Ь |
| 223 | Ы |
| 224 | а |
| 225 | б |
| ... | ... |
| 254 | ь |
| 255 | ы |

> ⚠️ **Ё / ё** находятся **вне основного блока**:  
> - `Ё` = **168 (0xA8)**  
> - `ё` = **184 (0xB8)**

---

### **4. Дополнительные символы (128–159)**

Это **главное преимущество** WIN1251. В этом диапазоне размещены часто используемые знаки:

| Код | Символ | Назначение |
|-----|--------|-----------|
| 131 | ƒ      | Функция (редко) |
| 132 | „      | Левые кавычки-«лапки» |
| 133 | …      | Многоточие |
| 134 | †      | Крест (сноска) |
| 135 | ‡      | Двойной крест |
| 138 | Љ      | Для сербского |
| 139 | ‹      | Одинарная левая кавычка |
| 140 | Њ      | Для сербского |
| 145 | ‘      | Левая одинарная кавычка |
| 146 | ’      | Правая одинарная кавычка |
| **147** | **“** | **Левые двойные кавычки** |
| **148** | **”** | **Правые двойные кавычки** |
| **150** | **–** | **Среднее тире (en dash)** |
| **151** | **—** | **Длинное тире (em dash)** |
| 152 | ˜      | Тильда |
| 153 | ™      | Торговая марка |
| 154 | љ      | Строчная для сербского |
| 155 | ›      | Одинарная правая кавычка |
| 156 | њ      | Строчная для сербского |
| **160** | ** ** | **Неразрывный пробел** |

> 💡 Именно наличие **«», «—», «…», «№»** (№ = 185) сделало WIN1251 **удобной для печати и делопроизводства**.

---

### **5. Сравнение с другими кодировками**

| Кодировка | Знаки препинания | Порядок кириллицы | Распространение |
|----------|------------------|------------------|----------------|
| **WIN1251** | ✅ Полный набор | Почти алфавитный | Очень высокое (Windows, веб) |
| **КОИ-8** | ❌ Минимум | Обратный | Высокое (UNIX, почта) |
| **ISO-8859-5** | ❌ Нет полезных знаков | Алфавитный | Низкое |
| **CP866** | ⚠️ Ограниченный | Алфавитный | Высокое (DOS) |

---

### **6. Использование в Интернете и ОС**

- **Windows**: основная кириллическая кодировка с 1990-х.
- **Веб**: доминировала до ~2010 г.  
  - Указывалась в HTML: `<meta charset="windows-1251">`
  - MIME-тип: `Content-Type: text/html; charset=windows-1251`
- **Базы данных**: часто использовалась как кодировка по умолчанию (MySQL, Oracle).
- **Программирование**: `Encoding.GetEncoding(1251)` в .NET.

> 🌐 Даже сегодня **миллионы старых сайтов** всё ещё работают в WIN1251.

---

### **7. Недостатки**

1. **Несовместимость с UNIX**:  
   - В 7-битном режиме кириллица превращается в **абракадабру** (в отличие от КОИ-8).
2. **Отсутствие стандарта ISO**:  
   - Это **частная кодировка Microsoft**, не утверждённая ISO.
3. **Ограниченность**:  
   - Поддерживает только **кириллицу и латиницу** — нет греческих, арабских, иероглифов.

---

### **8. WIN1251 в контексте БЭВМ-NG**

В методических указаниях по БЭВМ-NG (раздел 1.4.4) сказано:

> «Символьная информация хранится в **младших 8 битах 16-битного слова**».

Таким образом, текст в WIN1251 может быть представлен в БЭВМ-NG как последовательность слов:
- `'А'` → `0x00C0`
- `'а'` → `0x00E0`
- `'—'` (длинное тире) → `0x0097`
- `'№'` → `0x00B9`

Программист вручную управляет интерпретацией байтов — процессор не «знает» о кодировке.

### **10. Заключение**

**Windows-1251** — это **прагматичная, удобная и массово распространённая** кириллическая кодировка, разработанная с учётом **реальных потребностей пользователей**: типографика, делопроизводство, совместимость с Windows. Несмотря на отсутствие статуса международного стандарта, она стала **фактическим стандартом** для русскоязычного цифрового пространства на два десятилетия.

Сегодня её вытесняет **UTF-8**, но знание WIN1251 остаётся необходимым для работы с **легаси-системами, архивами и старыми веб-сайтами**.

> 📌 **Примечание**: В методических указаниях по БЭВМ-NG **WIN1251 не описывается**, так как документ фокусируется на архитектуре процессора, а не на кодировках.
