# 22. ALU, коммутатор, блок признаков результата

### **1. Общая структура обработки данных в БЭВМ-NG**

Согласно **разделу 1.2** методических указаний, процессор БЭВМ-NG включает три тесно взаимосвязанных функциональных блока, отвечающих за выполнение арифметико-логических операций:

1. **Арифметико-логическое устройство (АЛУ, ALU)** — выполняет основные операции.
2. **Коммутатор** — управляет пересылкой и преобразованием данных между АЛУ и регистрами.
3. **Блок установки признаков результата** — формирует флаги NZVC на основе результата операции.

Эти блоки работают **синхронно под управлением микропрограммного устройства (МПУ)** и обеспечивают полную функциональность команд типа `ADD`, `SUB`, `AND`, `ROL` и др.

---

### **2. Арифметико-логическое устройство (АЛУ)**

#### **2.1. Назначение**
Согласно тексту (раздел 1.2):

> «Арифметико-логическое устройство (АЛУ) может выполнять арифметические операции, такие как сложение и сложение с учётом переноса, полученного в результате выполнения предыдущей операции, операции логического умножения и инвертирования».

#### **2.2. Поддерживаемые операции**
- **Арифметические**:
  - Сложение (`A + B`)
  - Сложение с переносом (`A + B + C`)
  - Вычитание реализуется как `A + (~B) + 1`
- **Логические**:
  - И (`AND`)
  - ИЛИ (`OR`)
  - Инверсия (`NOT`)
- **Сдвиги**:
  - Циклические (`ROL`, `ROR`)
  - Арифметические (`ASL`, `ASR`)
- **Операции с флагами**: установка/сброс флагов `C`, `V`.

#### **2.3. Входы и выходы**
- **Левый вход**: подключён к аккумулятору (AC) или буферному регистру (BR) через коммутатор.
- **Правый вход**: подключён к регистру данных (DR) или к левому входу (для сдвигов).
- **Выход**: 16-битный результат поступает на вход коммутатора.

---

### **3. Коммутатор**

#### **3.1. Назначение**
Согласно методичке:

> «Коммутатор — это устройство, на которое поступают 18 разрядов результата операции из АЛУ (16-ти разрядный результат сложения и биты, необходимые для формирования признака переноса C), а также предыдущее значение переноса из регистра состояния. Коммутатор выполняет операции прямой и перекрёстной пересылки информации между разрядами слова, осуществляет арифметические и логические побитовые сдвиги влево и вправо, а также расширение знака младшего байта в старший».

#### **3.2. Функции коммутатора**
Коммутатор реализует следующие операции (таблица B.20):

| Сигнал | Операция | Описание |
|--------|---------|---------|
| `LTOL` | Прямая пересылка младшего байта | `AC[7:0] → AC[7:0]` |
| `LTOH` | Пересылка младшего → старший | `AC[7:0] → AC[15:8]` |
| `HTOL` | Пересылка старшего → младший | `AC[15:8] → AC[7:0]` |
| `HTOH` | Прямая пересылка старшего | `AC[15:8] → AC[15:8]` |
| `SEXT` | Расширение знака | `AC[7] → AC[15:8]` |
| `SHLT`, `SHRT` | Арифметические сдвиги | Знак сохраняется |
| `SHLO`, `SHRF` | Логические сдвиги | Вдвигается 0 |
| `ROL`, `ROR` | Циклические сдвиги | Через флаг `C` |

#### **3.3. Взаимодействие с АЛУ и регистрами**
Результат из АЛУ поступает в коммутатор, где может быть:
- записан в регистр (AC, DR и др.),
- снова подан на вход АЛУ (для многошаговых операций),
- проанализирован для установки флагов.

---

### **4. Блок установки признаков результата**

#### **4.1. Назначение**
> «Блок установки признаков результата предназначен для формирования однобитовых признаков результата, которые в конечном итоге сохраняются в младших 4-х битах регистра состояния».

#### **4.2. Флаги NZVC**
В БЭВМ-NG определены **четыре флага**, хранящиеся в регистре состояния (PS):

| Флаг | Название | Формирование |
|------|--------|-------------|
| **N** (Negative) | Знак | Копирует **15-й бит** результата (знак в дополнительном коде) |
| **Z** (Zero) | Ноль | Устанавливается, если результат **равен нулю** |
| **V** (oVerflow) | Переполнение | Формируется как **XOR переносов из 14-го и 15-го разрядов**: `V = C14 ⊕ C15` |
| **C** (Carry) | Перенос | Перенос из **старшего (15-го) разряда** АЛУ |

#### **4.3. Использование флагов**
Флаги используются **командами ветвления**:
- `BEQ`, `BNE` — проверка Z
- `BMI`, `BPL` — проверка N
- `BVS`, `BVC` — проверка V
- `BCS`, `BCC` — проверка C
- `BLT`, `BGE` — проверка N ⊕ V

Пример из таблицы B.3:
- Команда `ADD M` устанавливает **все флаги** (N, Z, V, C).
- Команда `AND M` устанавливает **N, Z**, сбрасывает **V = 0**, не влияет на **C**.

---

### **5. Пример выполнения команды ADD**

Рассмотрим команду `ADD 21` (адрес 21 содержит операнд).

Согласно **разделу 1.9** и рисункам B.8–B.9:

1. **IF**: команда `0x4021` загружается из памяти в CR.
2. **AF**: адрес `21` выделяется из команды → AR = 21.
3. **OF**: содержимое ячейки 21 → DR.
4. **EX**:  
   - AC → левый вход АЛУ,  
   - DR → правый вход АЛУ,  
   - АЛУ вычисляет `AC + DR`,  
   - результат → коммутатор,  
   - коммутатор → AC,  
   - блок флагов устанавливает **N, Z, V, C**.

---

### **6. Управление через микрокоманды**

В **микропрограмме** (таблица B.21) каждая операция управляется конкретными битами микрокоманды:

- `RDAC`, `RDDR` — подать AC и DR на входы АЛУ,
- `SORA` — выбрать операцию «сумма или ИЛИ»,
- `PLS1` — включить режим `A + B + 1` (для вычитания),
- `STNZ`, `SETC`, `SETV` — установить флаги.

Пример микрокоманды для `ADD` (адрес 32 в таблице B.21):
```
0010E09011 → RDAC, RDDR, SORA, STNZ, SETC, WRAC
```

---

### **7. Заключение**

ALU, коммутатор и блок признаков результата в БЭВМ-NG образуют **единый вычислительный комплекс**, который:
- выполняет все базовые арифметические и логические операции,
- поддерживает работу со знаковыми и беззнаковыми числами,
- обеспечивает управление программным потоком через флаги,
- управляется микропрограммным устройством с высокой степенью детализации.

Эта структура демонстрирует **реалистичную модель ALU реальных процессоров**, адаптированную для учебных целей.
