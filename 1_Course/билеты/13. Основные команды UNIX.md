# 13. Основные команды UNIX

### **1. Общая характеристика команд UNIX**

UNIX-команды — это **небольшие утилиты**, каждая из которых выполняет **одну конкретную задачу**. Эта философия («делай одну вещь и делай её хорошо») позволяет эффективно **комбинировать команды через конвейеры** (`|`) и перенаправления (`>`, `<`), создавая сложную логику обработки данных без написания программ.

Команды делятся на:
- **Встроенные (builtin)** — выполняются самой оболочкой (например, `cd`, `export`, `exit`).
- **Внешние** — отдельные исполняемые файлы (обычно в `/bin`, `/usr/bin`, `/sbin` и т.д.).

### **2. Классификация и подробное описание основных команд**

#### **2.1. Работа с файлами и каталогами**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `ls` | Список файлов и каталогов | `ls -l` — подробный список<br>`ls -a` — включая скрытые файлы<br>`ls -lh` — с удобочитаемыми размерами |
| `pwd` | Показать текущий рабочий каталог | `pwd` → `/home/user` |
| `cd` | Сменить каталог | `cd /etc` — перейти в `/etc`<br>`cd ..` — подняться на уровень выше<br>`cd ~` — домой |
| `mkdir` | Создать каталог | `mkdir dir`<br>`mkdir -p a/b/c` — рекурсивно |
| `rmdir` | Удалить **пустой** каталог | `rmdir empty_dir` |
| `rm` | Удалить файл(ы) или каталог(и) | `rm file`<br>`rm -r dir` — рекурсивно<br>`rm -f file` — без подтверждения |
| `cp` | Копировать файлы/каталоги | `cp a.txt b.txt`<br>`cp -r src/ dest/` |
| `mv` | Переместить или переименовать | `mv old.txt new.txt`<br>`mv file dir/` |

> 💡 **Важно**: В БЭВМ-NG **нет файлов, каталогов и этих команд**, так как отсутствует файловая система и операционная оболочка.

---

#### **2.2. Просмотр и редактирование текста**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `cat` | Вывести содержимое файла(ов) | `cat file`<br>`cat file1 file2 > combined` |
| `more` / `less` | Постраничный просмотр | `less log.txt` — прокрутка вверх/вниз, поиск (`/`) |
| `head` | Показать первые строки | `head -10 file` |
| `tail` | Показать последние строки | `tail -f /var/log/syslog` — «следить» за файлом |
| `echo` | Вывести строку | `echo "Hello"`<br>`echo $PATH` — подстановка переменной |
| `nano` / `vi` / `vim` | Текстовые редакторы | `vi config.txt` |

---

#### **2.3. Поиск и фильтрация**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `grep` | Поиск строк по шаблону | `grep "error" log.txt`<br>`grep -i "warn" file` — без учёта регистра<br>`grep -r "TODO" .` — рекурсивно |
| `find` | Поиск файлов по имени, размеру, времени и др. | `find /home -name "*.txt"`<br>`find . -type f -mtime -7` — изменённые за 7 дней |
| `which` | Найти путь к исполняемому файлу | `which python` → `/usr/bin/python` |
| `locate` | Быстрый поиск по индексу | `locate passwd` |

---

#### **2.4. Управление процессами**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `ps` | Список процессов | `ps aux` — все процессы<br>`ps -ef` — альтернативный формат |
| `top` / `htop` | Интерактивный мониторинг процессов | `htop` — цветной, с деревом |
| `kill` | Завершить процесс | `kill 1234` — по PID<br>`kill -9 1234` — принудительно |
| `pkill` / `killall` | Убить по имени | `pkill firefox` |
| `jobs`, `bg`, `fg`, `Ctrl+Z` | Управление заданиями (jobs) | `sleep 100` → `Ctrl+Z` → `bg` → `jobs` |

---

#### **2.5. Системная информация и управление**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `uname` | Информация о системе | `uname -a` — ядро, версия, архитектура |
| `df` | Свободное место на дисках | `df -h` — в удобочитаемом виде |
| `du` | Размер файлов и каталогов | `du -sh /var/log` — общий размер |
| `free` | Память (RAM и swap) | `free -h` |
| `who` / `w` | Кто вошёл в систему | `w` — с активностью |
| `date` / `cal` | Дата и календарь | `date` → `Tue Nov 25 10:00:00 MSK 2025` |

---

#### **2.6. Справка и документация**

| Команда | Назначение | Примеры |
|--------|-----------|--------|
| `man` | Руководство (manual) | `man ls` — описание команды<br>`man 5 passwd` — файл конфигурации |
| `--help` | Краткая справка | `ls --help` |
| `info` | Альтернатива `man` (более структурированная) | `info grep` |

### **3. Примеры составных команд (конвейеры)**

```bash
# Найти все запущенные процессы пользователя и отсортировать
ps aux | grep $USER | sort

# Подсчитать количество строк в .c файлах
wc -l *.c

# Найти все ошибки за сегодня в логе
grep "$(date '+%Y-%m-%d')" /var/log/syslog | grep "ERROR"

# Создать архив и сразу сжать
tar cf - /data | gzip > backup.tar.gz
```

---

### **4. Специальные файлы и пути**

- `/` — корень файловой системы
- `/home` — домашние каталоги пользователей
- `/etc` — конфигурационные файлы
- `/dev` — файлы устройств (например, `/dev/sda` — диск)
- `/proc` — виртуальная ФС с информацией о процессах
- `/tmp` — временные файлы

---

### **5. Заключение**

Освоение базовых UNIX-команд — фундамент для:
- системного администрирования,
- разработки (особенно в Linux-среде),
- анализа данных,
- автоматизации задач (через shell-скрипты).

Их сила — не в отдельных утилитах, а в **композиции**: простые команды, соединённые в конвейер, заменяют сложные программы.

> 📌 **Примечание**: В методических указаниях по **БЭВМ-NG** **UNIX-команды не используются и не упоминаются**, так как модель:
> - не имеет операционной системы,
> - не поддерживает файловую систему,
> - работает на уровне «голого железа» с 16-битными словами и машинными командами (типа `LD`, `ADD`, `JUMP`).

Вместо `cat`, `grep`, `ls` в БЭВМ-NG программист **вручную управляет памятью**, **читает/пишет через `IN`/`OUT`**, и **реализует логику через циклы и ветвления** на ассемблере БЭВМ.
