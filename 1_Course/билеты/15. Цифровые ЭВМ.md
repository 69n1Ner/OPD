# 15. Цифровые ЭВМ

### **1. Определение и суть цифровых ЭВМ**

**Цифровая ЭВМ (ЦВМ)** — это вычислительная машина, в которой **вся информация (данные и команды) представляется в дискретной (цифровой) форме**, обычно в виде **битов** (0 и 1). Все вычисления выполняются над **конечными последовательностями битов** с использованием **логических и арифметических операций**.

Основное отличие от аналоговых ЭВМ — **дискретность**:  
- нет непрерывных физических величин (напряжения, тока как аналога переменной),  
- есть **дискретные состояния** (логический 0 / 1),  
- работа строго **по программе**, хранящейся в памяти.

> ⚠️ **Важно**: В предоставленном файле **«Методические указания к выполнению лабораторных работ и рубежного контроля БЭВМ 2019 bcomp-ng v.1.45-1-сжатый.pdf»** термин «цифровая ЭВМ» **не используется напрямую**, но **вся модель БЭВМ-NG является цифровой** и описана в деталях. Поэтому основной корпус ответа — из методички.

### **2. Архитектура цифровой ЭВМ (на основе БЭВМ-NG)**

БЭВМ-NG — **простая цифровая ЭВМ аккумуляторного типа**, соответствующая классической архитектуре **фон Неймана** (см. раздел 1.2 методички).

#### **2.1. Основные компоненты БЭВМ-NG**

Согласно рис. B.1 и тексту (стр. 3–4):

1. **Память (RAM)**  
   - 2048 ячеек по 16 бит (адреса 0x000–0x7FF).  
   - Первые 256 ячеек (0x00–0xFF) зарезервированы под **векторы прерываний**.

2. **Процессор (Processor)**  
   Состоит из:
   - **Арифметико-логического устройства (АЛУ)** — выполняет:  
     - сложение, вычитание,  
     - логическое И, ИЛИ,  
     - сдвиги, инверсию.
   - **Коммутатора** — управляет пересылкой данных между регистрами и АЛУ, формирует флаги.
   - **Блока установки признаков результата** — устанавливает флаги **N, Z, V, C**.
   - **Устройства управления (УУ)** — микропрограммное (МПУ), управляет выполнением команд.

3. **Регистры (см. рис. B.1 и текст):**
   - **AC (Accumulator)** — 16-битный аккумулятор, основной операнд АЛУ.
   - **IP (Instruction Pointer)** — 11-битный счётчик команд.
   - **AR (Address Register)** — 11-битный регистр адреса.
   - **DR (Data Register)** — 16-битный буфер обмена с памятью.
   - **CR (Command Register)** — хранит текущую команду.
   - **BR (Buffer Register)** — временный буфер.
   - **SP (Stack Pointer)** — 11-битный указатель стека.
   - **PS (Program State)** — флаги и управляющие биты (режим «работа/останов», прерывания и др.).
   - **IR (Input Register)** — 16-битный регистр пульта оператора (ввод адреса/данных).

4. **Устройства ввода-вывода (ВУ)**  
   - ВУ-1 (вывод), ВУ-2 (ввод), ВУ-3 (ввод-вывод) и др. (см. раздел 2.1).
   - Обмен через команды `IN #reg`, `OUT #reg`.

5. **Пульт оператора**  
   - Клавиатура для ввода адреса/данных, кнопки «Пуск», «Такт», «Работа/Останов».

### **3. Принципы работы цифровой ЭВМ (по методичке)**

#### **3.1. Дискретное представление данных**
- Все данные — **16-битные слова**.
- Целые числа:  
  - **беззнаковые**: 0 … 65535,  
  - **знаковые**: в **дополнительном коде** (–32768 … +32767) (раздел 1.4.1).
- Логические данные — по битам.
- Символы — ASCII в младших 8 битах (раздел 1.4).

#### **3.2. Программируемость**
- Команды хранятся **в той же памяти**, что и данные (архитектура фон Неймана).
- Программа — последовательность **машинных команд** (табл. B.3).
- Команды 16-битные, делятся на 4 типа:
  1. **Безадресные**: `CLA`, `HLT`, `PUSH`, `POP` и др.
  2. **Ввод-вывод**: `IN`, `OUT`, `EI`, `DI`.
  3. **Адресные**: `LD M`, `ADD M`, `ST M`, `JUMP M`.
  4. **Ветвления**: `BEQ`, `BNE`, `BLT`, `BGE` и др.

#### **3.3. Цикл выполнения команды (раздел 1.9)**
Каждая команда выполняется за несколько **микроциклов** (микроопераций), управляемых **микропрограммным устройством (МПУ)**:
1. **IF (Instruction Fetch)** — выборка команды из памяти (`MEM[IP] → DR`).
2. **AF (Address Fetch)** — вычисление адреса операнда (если требуется).
3. **OF (Operand Fetch)** — загрузка операнда (`MEM[AR] → DR`).
4. **EX (Execution)** — выполнение операции в АЛУ.
5. **INT (Interruption)** — обработка прерываний.

> Пример для `ADD 21` (стр. 16–18 методички):  
> - Чтение команды из памяти → CR  
> - Адрес 21 → AR → чтение данных → DR  
> - `AC + DR → AC`  
> - Установка флагов NZVC.

#### **3.4. Стек и подпрограммы**
- Стек организован в верхней части памяти (адреса 0x7FF → 0x000).
- Команды: `CALL` (вызов), `RET` (возврат), `PUSH`/`POP` (сохранение регистров).
- Адрес возврата сохраняется в стеке (рис. B.6).

#### **3.5. Прерывания**
- Аппаратные (от ВУ) и программные (`INT #num`).
- Векторы прерываний — в ячейках 0x00–0xFF.
- Обработка: сохранение PS и IP в стек, переход по вектору, восстановление через `IRET`.

### **4. Преимущества цифровых ЭВМ**

1. **Высокая точность** — ошибки округления контролируемы (в отличие от аналоговых).
2. **Универсальность** — одна и та же машина решает любые задачи при смене программы.
3. **Воспроизводимость** — одинаковый результат при одних и тех же входных данных.
4. **Гибкость** — поддержка циклов, условий, рекурсии, сложных структур данных.
5. **Хранение программ и данных** — возможность многократного выполнения, отладки.

---

### **5. Историческое значение**

- **1940-е**: первые цифровые ЭВМ (ENIAC, EDVAC, EDSAC) заложили основы архитектуры фон Неймана.
- **1950–1970-е**: транзисторы → ИС → микропроцессоры → персональные компьютеры.
- **Современность**: все компьютеры — цифровые (ПК, серверы, смартфоны, встраиваемые системы).

---

### **6. Цифровая ЭВМ vs БЭВМ-NG**

БЭВМ-NG — **учебная модель цифровой ЭВМ**, демонстрирующая:
- Принципы фон Неймана (единая память для кода и данных),
- Машинный цикл (выборка → декодирование → выполнение),
- Работу с АЛУ, стеком, прерываниями,
- Микропрограммное управление.

Она не имеет ОС, файловой системы, графического интерфейса — только **«голое железо»**, что делает её идеальной для изучения основ архитектуры.

---

### **7. Заключение**

Цифровые ЭВМ — основа современных вычислительных систем. Их дискретная природа, программируемость и точность сделали их доминирующей парадигмой. Модель БЭВМ-NG, описанная в методических указаниях, является минималистичной, но полной реализацией этих принципов, позволяя глубоко понять работу процессора на уровне регистров, АЛУ и микрокода.
