### 4.1. Определение

**Ядро (_kernel_)** — центральный компонент UNIX/Linux, управляющий:

- процессами,
- памятью,
- устройствами ввода-вывода,
- файловыми системами,
- сетевыми соединениями.

### 4.2. Функции ядра

1. **Планировщик (scheduler)** — управляет распределением времени CPU между процессами.
2. **Менеджер памяти** — страничная организация, swap, copy-on-write.
3. **VFS (Virtual File System)** — абстрактный слой для поддержки разных ФС (ext4, NTFS, tmpfs).
4. **Сетевой стек** — реализация TCP/IP, UDP, raw sockets.
5. **Драйверы устройств** — загружаемые модули (`.ko` в Linux).

### 4.3. Особенности ядра Linux

- **Монолитное**, но с поддержкой **LKM (Loadable Kernel Modules)**.
- Реализует **POSIX API**.
- Поддержка **SMP (Symmetric Multiprocessing)**.
- Механизмы:
    - `slab allocator` — выделение памяти для объектов ядра,
    - `buddy system` — управление физической памятью,
    - `cgroups` — ограничение ресурсов,
    - `namespaces` — изоляция процессов.

### 4.4. Режимы работы

- **Ядро (kernel mode)** — полный доступ к аппаратуре.
- **Пользовательский (user mode)** — ограниченные привилегии.
- Переход через **системные вызовы** (`syscall`).

> 📌 **Примечание**: В методичке по БЭВМ **ядро не рассматривается**, так как модель не включает ОС.
