# 75. Сетевой уровень IP

### **1. Место сетевого уровня в модели OSI/TCP/IP**

**Сетевой уровень** — это **третий уровень** модели OSI и **второй уровень** модели TCP/IP (там он называется **Internet Layer**).

**Основная функция**: обеспечение **межсетевой передачи данных** — то есть маршрутизация пакетов **между разными сетями** (в отличие от канального уровня, который работает только **внутри одной сети**).

**Основной протокол сетевого уровня** — **IP (Internet Protocol)**, существующий в двух версиях: **IPv4** и **IPv6**.

---

### **2. Основные функции IP**

1. **Логическая адресация** — присвоение каждому устройству **уникального IP-адреса**.
2. **Маршрутизация** — выбор пути от источника к получателю через промежуточные маршрутизаторы.
3. **Фрагментация и сборка** — разбиение больших пакетов на части, если MTU (Maximum Transmission Unit) сети меньше размера пакета.
4. **Доставка без установления соединения** — каждый пакет обрабатывается **независимо**.
5. **Ненадёжная доставка** — IP **не гарантирует**:
   - доставку пакета,
   - порядок следования пакетов,
   - отсутствие дубликатов.

> ⚠️ Надёжность обеспечивается **транспортными протоколами** (например, TCP).

---

### **3. Структура IP-пакета (IPv4)**

IPv4-пакет состоит из **заголовка** и **данных** (обычно TCP/UDP-сегмент).

| Поле | Размер | Описание |
|------|--------|---------|
| **Версия** | 4 бита | `4` = IPv4, `6` = IPv6 |
| **Длина заголовка (IHL)** | 4 бита | В 32-битных словах (обычно `5` → 20 байт) |
| **Тип обслуживания (ToS / DSCP)** | 8 бит | Приоритет, QoS (например, для VoIP) |
| **Общая длина** | 16 бит | Длина всего пакета в байтах (макс. 65 535) |
| **Идентификатор** | 16 бит | Уникальный ID для фрагментации |
| **Флаги** | 3 бита | `MF` (More Fragments), `DF` (Don’t Fragment) |
| **Смещение фрагмента** | 13 бит | Позиция фрагмента в исходном пакете |
| **Время жизни (TTL)** | 8 бит | Макс. число маршрутизаторов на пути (уменьшается на 1 на каждом) |
| **Протокол верхнего уровня** | 8 бит | `6` = TCP, `17` = UDP, `1` = ICMP |
| **Контрольная сумма заголовка** | 16 бит | Для обнаружения ошибок в заголовке |
| **IP-адрес источника** | 32 бита | Адрес отправителя |
| **IP-адрес назначения** | 32 бита | Адрес получателя |
| **Опции (опционально)** | переменная | Диагностика, маршрутизация по списку |
| **Данные** | переменная | Полезная нагрузка (TCP/UDP/ICMP и т.д.) |

> 💡 **Минимальный заголовок IPv4 = 20 байт** (если нет опций).

---

### **4. IP-адресация (IPv4)**

- **Длина**: 32 бита → записывается в виде **четырёх десятичных чисел** (октетов): `192.168.1.10`.
- **Классовая адресация** (устарела): классы A, B, C, D, E.
- **Бесклассовая адресация (CIDR)** — современный стандарт:
  - Запись: `192.168.1.0/24`
  - `/24` = 24 бита под сеть, 8 бит под хосты → 256 адресов.
- **Типы адресов**:
  - **Unicast** — один получатель,
  - **Broadcast** — все хосты в сети (`192.168.1.255/24`),
  - **Multicast** — группа получателей (`224.0.0.0` – `239.255.255.255`).

---

### **5. Маршрутизация**

- **Маршрутизатор** — устройство сетевого уровня, соединяющее разные сети.
- Использует **таблицу маршрутизации**, содержащую:
  - Сеть назначения,
  - Маску подсети,
  - Шлюз (next hop),
  - Интерфейс вывода.
- Алгоритмы маршрутизации:
  - **Статическая** — администратор вручную задаёт маршруты.
  - **Динамическая** — протоколы обмениваются информацией:
    - **RIP** (Routing Information Protocol) — дистанционно-векторный,
    - **OSPF** (Open Shortest Path First) — по состоянию каналов,
    - **BGP** (Border Gateway Protocol) — между автономными системами (в Интернете).

---

### **6. Фрагментация**

Если пакет больше MTU сети (например, 1500 байт в Ethernet), он **фрагментируется**:

- Каждый фрагмент содержит:
  - Заголовок IP (копия оригинального, с изменениями),
  - Часть данных.
- Поля **Идентификатор**, **Смещение**, **MF** позволяют **собрать пакет** на приёмнике.
- **Фрагментация снижает производительность** и **увеличивает риск потери** (при потере одного фрагмента теряется весь пакет).

> 💡 В **IPv6 фрагментация разрешена только на источнике**, а не на промежуточных маршрутизаторах.

---

### **7. Вспомогательные протоколы сетевого уровня**

| Протокол | Функция |
|--------|--------|
| **ICMP (Internet Control Message Protocol)** | Диагностика и сообщения об ошибках (`ping`, `traceroute`, «сеть недоступна») |
| **ARP (Address Resolution Protocol)** | Преобразование **IP → MAC** в локальной сети (только IPv4) |
| **NDP (Neighbor Discovery Protocol)** | Аналог ARP в **IPv6**, плюс автонастройка адресов |
| **IGMP (Internet Group Management Protocol)** | Управление **multicast-группами** |

---

### **8. IPv6 — следующее поколение IP**

- **Адрес**: 128 бит → `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
- **Преимущества**:
  - Огромное адресное пространство (~3.4×10³⁸ адресов),
  - Упрощённый заголовок (фикс. 40 байт),
  - Встроенная поддержка **безопасности (IPsec)**,
  - Автоконфигурация (SLAAC),
  - Отсутствие фрагментации на маршрутизаторах.

---

### **9. Заключение**

IP — это **основа Интернета**, обеспечивающая **универсальную маршрутизацию** между миллиардами устройств. Его простота, масштабируемость и независимость от физической среды сделали его **де-факто стандартом сетевого уровня**.

> 📌 **Примечание**: В методических указаниях по БЭВМ-NG **IP и сетевой уровень не упоминаются**, так как БЭВМ — учебная модель процессора без сетевых функций.
