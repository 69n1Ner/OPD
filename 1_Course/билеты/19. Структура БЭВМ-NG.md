# 19. Структура БЭВМ-NG

### **1. Общая характеристика модели**

Согласно разделу 1.1 методических указаний:

> «Базовая ЭВМ — это простая модельная машина, отражающая типичные черты многих конкретных ЭВМ... На начинать изучение целесообразно с подобных моделей, используя ее модели, построенные для различных типов персональных ЭВМ».

Таким образом, **БЭВМ-NG — не реальная ЭВМ, а учебная модель**, созданная для демонстрации основных принципов архитектуры цифровых вычислительных машин. Она представляет собой **аккумуляторную ЭВМ**, работающую с 16-битными словами, и реализует классическую **архитектуру фон Неймана** (единая память для кода и данных).

---

### **2. Основные компоненты БЭВМ-NG (по рисунку B.1)**

На рисунке B.1 представлена упрощенная структурная схема БЭВМ-NG. Все функциональные блоки соединены системной шиной и управляются устройством управления (УУ). Основные компоненты:

#### **2.1. Процессор (Processor)**
Процессор БЭВМ-NG — это центральный блок, состоящий из:
- **Арифметико-логического устройства (АЛУ)** — выполняет арифметические и логические операции.
- **Коммутатора** — управляет пересылкой данных между регистрами и АЛУ.
- **Блока установки признаков результата** — формирует флаги NZVC.
- **Регистров**: AC, IP, AR, DR, CR, BR, SP, PS, IR — хранят данные, адреса, команды и состояние процессора.

#### **2.2. Память (RAM)**
- Объем: **2048 ячеек** по **16 бит** (адреса 0x000–0x7FF).
- Первые 256 ячеек (0x00–0xFF) зарезервированы под **векторы прерываний**.
- Единое адресное пространство — архитектура фон Неймана.

#### **2.3. Устройство управления (УУ)**
- Реализовано как **микропрограммное устройство (МПУ)**.
- Выполняет машинные команды, разбивая их на последовательность микрокоманд.
- Управляет всеми остальными блоками.

#### **2.4. Устройства ввода-вывода (ВУ)**
- ВУ-1 (вывод), ВУ-2 (ввод), ВУ-3 (ввод-вывод) и др. (в новых версиях — таймер, текстовый принтер, 7-сегментный индикатор и т.д.).
- Обмен данными осуществляется через команды `IN` и `OUT`.

#### **2.5. Пульт оператора**
- Клавиатура для ввода адреса/данных.
- Кнопки «Пуск», «Такт», «Работа/Останов».
- Индикаторы состояния.

---

### **3. Регистры процессора (подробное описание)**

Согласно рисунку B.1 и таблице B.20, процессор БЭВМ-NG содержит следующие регистры:

| Регистр | Назначение | Разрядность |
|---------|------------|-----------|
| **AC (Accumulator)** | Аккумулятор — основной регистр для арифметико-логических операций. | 16 бит |
| **IP (Instruction Pointer)** | Счетчик команд — указывает на следующую выполняемую команду. | 11 бит (адресное пространство 2¹¹ = 2048 ячеек) |
| **AR (Address Register)** | Регистр адреса — хранит адрес операнда в памяти. | 11 бит |
| **DR (Data Register)** | Регистр данных — буфер обмена с памятью. | 16 бит |
| **CR (Command Register)** | Регистр команды — хранит текущую машинную команду. | 16 бит |
| **BR (Buffer Register)** | Буферный регистр — временно хранит данные во время работы. | 16 бит |
| **SP (Stack Pointer)** | Указатель стека — указывает на вершину стека. | 11 бит |
| **PS (Program State)** | Регистр состояния — хранит флаги NZVC и управляющие биты (разрешение прерываний, режим работы). | 16 бит (используются 9 младших) |
| **IR (Input Register)** | Регистр пульта оператора — хранит введенный адрес или данные. | 16 бит |

### **4. Архитектура фон Неймана в БЭВМ-NG**

БЭВМ-NG полностью соответствует архитектуре фон Неймана:
- **Единая память** для кода и данных.
- **Цикл выполнения команды**: выборка → декодирование → выполнение.
- **Программируемость** — изменение поведения ЭВМ достигается заменой программы в памяти.

> 🔍 Пример из методички (стр. 20):  
> Программа `Z = –X + Y` размещается в памяти так:  
> - Адрес 0x020: данные X = –5 (0xFFAD)  
> - Адрес 0x021: данные Y = 106 (0x0106)  
> - Адрес 0x022: место для результата Z  
> - Адрес 0x023: команда `CLA`  
> - Адрес 0x024: команда `SUB 20`  
> Это прямое доказательство единой памяти для кода и данных.

### **5. Микропрограммное управление (МПУ)**

Устройство управления в БЭВМ-NG реализовано как **микропрограммное устройство (МПУ)**, что означает:
- Каждая машинная команда (например, `ADD M`) разбивается на последовательность **микрокоманд**.
- Микрокоманды хранятся в **памяти микрокоманд** (в эмуляторе — в отдельной области памяти).
- Выполнение управляется **счетчиком микрокоманд (СЧМК)**.

#### **Формат микрокоманды (рис. B.18, табл. B.20)**
Микрокоманда — 40-битное слово, где каждый бит (или группа битов) управляет отдельной линией управления.

##### **Операционная микрокоманда (ОМК)**:
```
[0–15] — управление чтением регистров (RDAC, RDDR и др.)
[16–23] — управление преобразованиями и установкой флагов (SEXT, SETC и др.)
[24–31] — управление записью в регистры (WRAC, WRDR и др.)
[32–33] — взаимодействие с памятью: LOAD (чтение), STOR (запись)
[34–35] — управление ВУ: IO (ввод-вывод)
[38] — HALT (останов)
[39] — TYPE = 0 (указывает, что это ОМК)
```

##### **Управляющая (ветвящая) микрокоманда (УМК)**:
```
[16–23] — номер бита для проверки (в данных с коммутатора)
[24–31] — адрес перехода (куда перейти, если бит = 1)
[32] — тип сравнения: прямой или инверсный
[39] — TYPE = 1 (указывает, что это УМК)
```

### **6. Цикл выполнения команды**

Согласно разделу 1.9, выполнение любой машинной команды состоит из **пяти микроциклов**, управляемых МПУ:

1. **IF (Instruction Fetch)** — выборка команды из памяти по адресу, указанному в IP.
2. **AF (Address Fetch)** — вычисление адреса операнда (если требуется, с учётом режима адресации).
3. **OF (Operand Fetch)** — загрузка операнда из памяти в DR.
4. **EX (Execution)** — выполнение операции в АЛУ.
5. **INT (Interruption)** — проверка запросов на прерывание и их обработка (если разрешены).

Каждый микроцикл реализуется несколькими микрокомандами, записанными в интерпретаторе БЭВМ (приложение B, таблица B.21).

---

### **7. Стек и подпрограммы**

Стек организован в **старших адресах** памяти (начиная с 0x7FF и растёт вниз). Указатель стека (SP) — 11-битный регистр.

- **Команды работы со стеком**: `PUSH`, `POP`, `CALL`, `RET`, `IRET`.
- **Команда `CALL M`**:
  1. Сохраняет `IP` и `PS` в стеке (`SP = SP – 1`, `SP = SP – 1`).
  2. Переходит к подпрограмме по адресу `M`.
- **Команда `RET`**:
  1. Восстанавливает `IP` из стека (`SP = SP + 1`).
  2. Восстанавливает `PS` из стека (`SP = SP + 1`).

---

### **8. Прерывания**

БЭВМ-NG поддерживает аппаратные и программные прерывания.

- **Векторы прерываний** — хранятся в первых 256 ячейках памяти (0x00–0xFF).
- **Обработка прерываний**:
  1. Сохранение `IP` и `PS` в стеке.
  2. Переход по вектору прерывания.
  3. Выполнение подпрограммы обработки.
  4. Возврат из прерывания (`IRET`).

---

### **9. Заключение**

Структура БЭВМ-NG — это **минимальная, но полная модель цифровой ЭВМ**, демонстрирующая ключевые принципы:
- архитектуру фон Неймана,
- микропрограммное управление,
- работу с памятью, регистрами и стеком,
- поддержку прерываний.

Эта модель идеально подходит для обучения основам архитектуры ЭВМ, так как она **прозрачна, детализирована и полностью описана в методических указаниях**.
