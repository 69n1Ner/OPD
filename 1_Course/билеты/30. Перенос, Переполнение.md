### **30. Перенос, Переполнение**

При арифметических операциях над \(n\)-битными числами используются два флага:

- **C (Carry)** — флаг **переноса** → для **беззнаковой** арифметики,  
- **V (Overflow)** — флаг **переполнения** → для **знаковой** арифметики (дополнительный код).

#### **Флаг C (Carry)**
- Устанавливается при **переносе из старшего (n–1)-го бита**.
- При сложении: \(C = 1\), если \(A + B \geq 2^n\).  
- При вычитании: \(C = 0\), если \(A < B\) (беззнаковое сравнение).
- Используется в командах:  
  - `BCC` — переход, если \(A \geq B\) (беззнак.),  
  - `BCS` — переход, если \(A < B\) (беззнак.).

#### **Флаг V (Overflow)**
- Устанавливается, если результат **выходит за знаковый диапазон** \([-2^{n-1}, 2^{n-1}-1]\).
- Возникает при:
  - (+) + (+) → (–),  
  - (–) + (–) → (+).
- Вычисляется как:  
  $$
  V = C_{n-2} \oplus C_{n-1}
  $$
  (исключающее ИЛИ между переносом **в** и **из** старшего бита).
- Используется в командах:  
  - `BLT`, `BGE` — знаковое сравнение,  
  - `BVS` — переход при переполнении.

#### **Сравнение**
| Флаг | Арифметика | Фиксирует | Пример ошибки |
|------|------------|-----------|----------------|
| **C** | Беззнаковая | Выход за \(0 … 2^n - 1\) | \(65535 + 1 = 0\) |
| **V** | Знаковая | Выход за \(-2^{n-1} … 2^{n-1}-1\) | \(32767 + 1 = -32768\) |

> **Важно**:  
> - Беззнаковая арифметика → смотрим **C**, игнорируем V.  
> - Знаковая арифметика → смотрим **V**, игнорируем C.


# 30. Перенос, Переполнение

### **1. Общая цель флагов C и V**

При выполнении арифметических операций над числами фиксированной разрядности (например, 16 бит в БЭВМ-NG) результат может **не поместиться** в отведённое количество бит. Для отражения этого процессор устанавливает два ключевых флага:

- **C (Carry)** — **флаг переноса**,
- **V (Overflow)** — **флаг переполнения**.

Они **не дублируют друг друга** — каждый отвечает за **разный тип арифметики**:
- **C** — для **беззнаковых чисел**,
- **V** — для **знаковых чисел (в дополнительном коде)**.

---

### **2. Флаг переноса (C — Carry)**

#### **2.1. Определение**
Флаг **C** устанавливается, если при сложении (или вычитании) возникает **перенос из старшего (знакового) разряда**.

- При **сложении**:  
  $$
  C = 1, \text{ если } (A + B) \geq 2^n
  $$
- При **вычитании**:  
  $$
  C = 0, \text{ если } A < B \text{ (в беззнаковой интерпретации)}
  $$

> 💡 В БЭВМ флаг **C** отражает **перенос из 15-го бита** (для 16-битных слов).

#### **2.2. Пример (16 бит, беззнаковая арифметика)**
- `0xFFFF + 0x0001 = 0x10000`  
  → Результат не помещается в 16 бит → **C = 1**  
  → Интерпретация:  
  - Беззнаковая: `65535 + 1 = 0` с переносом → корректно по модулю \(2^{16}\),
  - Знаковая: `–1 + 1 = 0` → корректно, но **C здесь не важен**.

#### **2.3. Использование**
Флаг **C** используется в командах беззнакового сравнения:
- `BCC` — Branch if Carry Clear → «если A ≥ B (беззнаковое)»,
- `BCS` — Branch if Carry Set → «если A < B (беззнаковое)».

---

### **3. Флаг переполнения (V — Overflow)**

#### **3.1. Определение**
Флаг **V** устанавливается, если при сложении/вычитании **знаковых чисел** истинный результат **выходит за пределы диапазона** \([-2^{n-1}, 2^{n-1}-1]\).

Это происходит **только при одинаковых знаках операндов и противоположном знаке результата**.

> ✅ Правило:  
> - `(+A) + (+B) = отрицательный` → **V = 1**,  
> - `(-A) + (-B) = положительный` → **V = 1**.

#### **3.2. Формула вычисления**
$$
V = C_{14} \oplus C_{15}
$$
где:
- \(C_{14}\) — перенос **в** старший (15-й) бит,
- \(C_{15}\) — перенос **из** старшего бита.

Если переносы **различаются** → произошло **знаковое переполнение**.

#### **3.3. Пример (16 бит, знаковая арифметика)**
- `0x7FFF + 0x0001 = 0x8000`  
  → `+32767 + 1 = –32768`  
  → Результат **некорректен** для знаковых чисел → **V = 1**  
  → Но **C = 0** (нет переноса из 15-го бита в 16-й).

#### **3.4. Использование**
Флаг **V** используется в командах знакового сравнения:
- `BVS` — Branch if Overflow Set → «произошло переполнение»,
- В паре с **N** — для корректного сравнения:  
  `BLT` (Branch if Less Than) = `(N ⊕ V) = 1`.

---

### **4. Сравнение C и V**

| Характеристика | **C (Carry)** | **V (Overflow)** |
|----------------|--------------|------------------|
| **Тип арифметики** | Беззнаковая | Знаковая (доп. код) |
| **Что фиксирует** | Перенос из старшего бита | Несоответствие знака результата |
| **Диапазон** | \(0 … 2^n - 1\) | \(-2^{n-1} … 2^{n-1} - 1\) |
| **Пример ошибки** | `65535 + 1 = 0` | `32767 + 1 = –32768` |
| **Используется в** | `BCC`, `BCS` | `BLT`, `BGE`, `BVS` |

> ⚠️ **Важно**:  
> - При беззнаковой арифметике **игнорируем V**, смотрим на **C**,  
> - При знаковой арифметике **игнорируем C**, смотрим на **V**.

---

### **5. Перенос и переполнение в БЭВМ-NG**

Согласно **разделу 1.2** методических указаний:
- После выполнения арифметических команд (`ADD`, `SUB`, `INC`, `DEC` и др.) устанавливаются флаги **C** и **V**,
- Флаги хранятся в **регистре признаков (PS)**,
- Команды ветвления (`BEQ`, `BCC`, `BVS` и др.) используют эти флаги для управления потоком программы.

> 💡 Пример из методички:  
> После `ADD` процессор автоматически выставляет C и V, и программист выбирает, **какую интерпретацию использовать** — знаковую или беззнаковую — через соответствующие команды ветвления.

---

### **6. Заключение**

Флаги **переноса (C)** и **переполнения (V)** — это **два взгляда на одну и ту же операцию сложения/вычитания**:
- **C** говорит: «Результат вышел за пределы беззнакового диапазона»,
- **V** говорит: «Результат вышел за пределы знакового диапазона».

Понимание разницы между ними **критически важно** для написания корректных программ на ассемблере, особенно при работе с числами, близкими к границам диапазона. В БЭВМ-NG эта двойственность реализована явно и позволяет гибко управлять логикой программы через систему флагов и ветвлений.

> 📚 **Источник**: раздел 1.2 методических указаний по БЭВМ-NG.
