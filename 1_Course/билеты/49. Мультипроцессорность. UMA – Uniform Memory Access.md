### **49. Мультипроцессорность: UMA (Uniform Memory Access)**
**UMA** — архитектура симметричной мультипроцессорной системы (SMP), в которой **все процессоры имеют одинаковое время доступа ко всей оперативной памяти**.
#### **Основные черты**
- Единое физическое адресное пространство,
- Все CPU равноправны (симметрия),
- Память общая и централизованная,
- Доступ к любой ячейке памяти занимает **одинаковое время** для любого процессора.
#### **Аппаратная реализация**
- Процессоры подключены к общей памяти через **общую шину, кроссбар или коммутатор**,
- Требуется **аппаратная когерентность кэшей** (протоколы MESI/MOESI).
#### **Преимущества**
- Простота программирования (единое адресное пространство),
- Эффективность при частом обмене данными между потоками,
- Низкие задержки доступа к памяти.
#### **Недостатки**
- **Узкое место** — общая память и шина,
- **Ограниченная масштабируемость** (обычно до 8–16 ядер),
- Конкуренция за пропускную способность памяти при росте числа CPU.
#### **Сравнение с NUMA**
| Параметр | UMA | NUMA |
|---------|-----|------|
| Время доступа к памяти | Одинаковое | Зависит от близости CPU к банку памяти |
| Масштабируемость | Низкая | Высокая |
| Сложность программирования | Низкая | Высокая |

# 49. Мультипроцессорность: UMA – Uniform Memory Access

### **1. Определение**

**UMA (Uniform Memory Access)** — это архитектура **симметричной мультипроцессорной системы (SMP)**, в которой **все процессоры имеют равный (одинаковый по времени) доступ ко всей оперативной памяти** через **общую шину или коммутатор**.

> 💡 Ключевая идея: **память — единый ресурс**, и **нет «своей» памяти у каждого процессора**.

---

### **2. Структура UMA-системы**

```
    +--------+     +--------+     +--------+
    | CPU 0  |     | CPU 1  |     | CPU N  |
    +--------+     +--------+     +--------+
          \             |             /
           \            |            /
            \           |           /
             \          |          /
              \         |         /
               \        |        /
                \       |       /
                 \      |      /
                  \     |     /
                   \    |    /
                    \   |   /
                     \  |  /
                      \ | /
                   +----------+
                   | Общая    |
                   | память   |
                   | (RAM)    |
                   +----------+
```

- Все процессоры подключены к **единой общей памяти**,
- Доступ к любой ячейке памяти занимает **одинаковое время** для любого процессора,
- Используется **общая шина, кроссбар или коммутатор** для соединения.

---

### **3. Основные характеристики**

| Характеристика | Описание |
|----------------|---------|
| **Симметричность** | Все процессоры равноправны, выполняют одинаковые задачи |
| **Единое адресное пространство** | Программа видит одну непрерывную память |
| **Когерентность кэша** | Требуется механизм (например, протокол MESI) для согласования данных в кэшах разных CPU |
| **Масштабируемость** | Ограничена пропускной способностью общей шины/памяти (обычно до 8–16 ядер) |

---

### **4. Преимущества UMA**

- **Простота программирования**: разработчик работает с **единым адресным пространством**, не думая о локальности данных.
- **Легкое распределение задач**: любая задача может быть назначена любому процессору.
- **Эффективность при разделяемых данных**: если процессы часто обмениваются данными, UMA минимизирует задержки.
- **Низкая задержка доступа к памяти** (по сравнению с NUMA для «удалённых» данных).

---

### **5. Недостатки**

- **Узкое место — общая память и шина**: при увеличении числа процессоров они начинают **конкурировать за доступ к памяти**.
- **Ограниченная масштабируемость**: производительность перестаёт расти после ~8–16 ядер из-за конфликтов на шине.
- **Высокая нагрузка на контроллер памяти**.

> ⚠️ Именно поэтому UMA используется в **настольных ПК и серверах начального уровня**, но не в крупных вычислительных системах.

---

### **6. Примеры реализации**

- **Многоядерные процессоры Intel/AMD (до 2 сокетов)**:  
  Все ядра делят **общий контроллер памяти и кэш L3** → классическая UMA.
- **Серверные платформы с одним сокетом**:  
  Например, Intel Xeon W, AMD Ryzen Threadripper.
- **Ранние SMP-системы**:  
  Sun Ultra Enterprise, HP 9000 — использовали общую шину.

> 💡 Даже в многосокетных системах **внутри одного сокета** часто используется UMA.

---

### **7. UMA vs NUMA**

| Параметр | **UMA** | **NUMA** |
|---------|--------|---------|
| Доступ к памяти | Равномерный для всех CPU | Зависит от близости CPU к банку памяти |
| Масштабируемость | Низкая (до 16 ядер) | Высокая (сотни ядер) |
| Сложность программирования | Низкая | Высокая (требуется учёт локальности) |
| Производительность при разделяемых данных | Высокая | Может страдать из-за межузловых обменов |

---

### **8. UMA и операционные системы**

Современные ОС (Linux, Windows) поддерживают UMA «из коробки»:
- Планировщик задач равномерно распределяет потоки по ядрам,
- Менеджер памяти выделяет страницы без привязки к CPU,
- Когерентность кэша обеспечивается **аппаратно** (протоколы MESI, MOESI).

---

### **9. Заключение**

UMA — это **простая, эффективная и удобная архитектура** для систем с небольшим числом процессоров. Она обеспечивает **единое адресное пространство и предсказуемую производительность**, что делает её идеальной для настольных компьютеров, рабочих станций и небольших серверов. Однако из-за фундаментального ограничения — **конкуренции за общую память** — она уступает место NUMA в крупномасштабных системах.
