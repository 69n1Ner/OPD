[[6. Права доступа к файлам в NIX]]
### **7.1. Символьный (буквенный) способ**

Использует команду `chmod` с указанием:

- Категории: `u`, `g`, `o`, `a` (все).
- Операции: `+` (добавить), `-` (убрать), `=` (установить).
- Права: `r`, `w`, `x`.

**Примеры:**
`chmod 755 script.sh   # rwxr-xr-x
`chmod u+x file        # добавить владельцу право на выполнение
`chmod go-w file       # запретить запись группе и остальным


chmod g+w file # разрешить запись группе

chmod o-rx file # запретить чтение и выполнение остальным

chmod a=r file # разрешить только чтение всем

### **7.2. Восьмеричный (числовой) способ**

Каждое право кодируется битом:

- `r` = 4
- `w` = 2
- `x` = 1

Сумма даёт цифру от 0 до 7. Три цифры задают права для владельца, группы и остальных.

**Примеры:**

- `chmod 755 file` → `rwxr-xr-x`
- `chmod 644 file` → `rw-r--r--`
- `chmod 600 file` → `rw-------` (только владелец)

### **7.3. Установка специальных битов**

- setuid: `chmod u+s file` или `chmod 4755 file`
- setgid: `chmod g+s file` или `chmod 2755 file`
- sticky bit: `chmod +t /tmp` или `chmod 1777 /tmp`

### **7.4. Проверка прав**

Команда `ls -l` отображает права в символьной форме:

bash

`-rwxr-xr-- 1 user group 1024 Jan 1 file

> 📌 **В БЭВМ-NG**: **нет прав доступа**, так как нет файлов, пользователей и inode.
