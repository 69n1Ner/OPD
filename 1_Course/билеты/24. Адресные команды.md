# 24. Адресные команды

### **1. Определение и назначение**

Согласно **разделу 1.3** методических указаний:

> «Адресные команды предписывают машине производить действия с ячейкой памяти, адрес которой определяется исходя из адресной части команды, состоящей из 12 бит (биты 0…11). КО (биты 12…15) принимает значения от 0x2 до 0xE и задаёт операцию».

Таким образом, **адресные команды** — это машинные инструкции, **явно использующие данные из памяти**, причём адрес операнда **кодируется непосредственно в теле команды** (в младших 12 битах).

---

### **2. Формат адресных команд**

Адресные команды делятся на **два подтипа**, в зависимости от значения **11-го бита** (см. рис. B.2в и рис. B.2г):

#### **2.1. Абсолютная адресация**  
- **Бит 11 = 0**  
- **Биты 0–10** содержат **11-битный абсолютный адрес** (0x000–0x7FF).  
- Процессор обращается к ячейке **напрямую** по этому адресу.

> Пример:  
> `ADD 21` → машинный код `0x4021`  
> - КО = `0100` (4) → команда ADD  
> - Бит 11 = `0` → абсолютная адресация  
> - Адрес = `00000010101₂ = 21₁₀`

#### **2.2. Относительная адресация**  
- **Бит 11 = 1**  
- **Биты 8–10** задают **режим адресации** (см. табл. B.10),  
- **Биты 0–7** содержат **8-битное смещение** (от –128 до +127).

Режимы относительной адресации:

| Биты 8–10 | Режим | Описание |
|----------|------|---------|
| `000` | Прямая относительная | `IP + смещение` |
| `001` | Зарезервировано | — |
| `010` | Косвенная автоинкрементная | Загрузка по адресу, затем `адрес := адрес + 1` |
| `011` | Косвенная автодекрементная | Сначала `адрес := адрес – 1`, затем загрузка |
| `100` | Относительно SP | `SP + смещение` (доступ к параметрам в стеке) |
| `101` | Зарезервировано | — |
| `110` | Прямая относительная (дубль `000`) | То же, что `000` |
| `111` | Непосредственная загрузка | Смещение интерпретируется как **8-битное знаковое число**, расширяется до 16 бит и используется **как операнд, а не адрес** |

> Пример:  
> `LD #5` → `0xB005`  
> - КО = `1011` (B) → LD  
> - Бит 11 = `1`  
> - Биты 8–10 = `111` → непосредственная загрузка  
> - Смещение = `00000101₂ = 5` → расширение знака → `0x0005` → `AC = 5`

> ⚠️ **Важно**: Несмотря на то, что непосредственная загрузка использует **формат адресной команды**, она **не обращается к памяти**. Это исключение, обусловленное экономией форматов команд.

### **3. Список адресных команд (табл. B.3)**

| КО | Команда | Описание | Флаги (N, Z, V, C) |
|----|--------|----------|---------------------|
| `2XXX` | `AND M` | `AC & M → AC` | N*, Z*, V=0 |
| `3XXX` | `OR M` | `AC \| M → AC` | N*, Z*, V=0 |
| `4XXX` | `ADD M` | `AC + M → AC` | N*, Z*, V*, C* |
| `5XXX` | `ADC M` | `AC + M + C → AC` | N*, Z*, V*, C* |
| `6XXX` | `SUB M` | `AC - M → AC` | N*, Z*, V*, C* |
| `7XXX` | `CMP M` | Сравнение `AC - M` | N*, Z*, V*, C* (результат **не сохраняется**) |
| `8XXX` | `LOOP M` | `M := M - 1`; если `M ≤ 0`, то `IP := IP + 1` | — |
| `AXXX` | `LD M` | `M → AC` | N*, Z*, V=0 |
| `BXXX` | `SWAM M` | Обмен `AC ↔ M` | N*, Z*, V=0 |
| `CXXX` | `JUMP M` | `M → IP` | — |
| `DXXX` | `CALL M` | Вызов подпрограммы по адресу `M` | — |
| `EXXX` | `ST M` | `AC → M` | — |

> Примечание: `*` — флаг устанавливается по результату, `0` — сбрасывается, `—` — не изменяется.

---

### **4. Особенности выполнения адресных команд**

#### **4.1. Цикл выполнения**
Согласно **разделу 1.9**, выполнение адресной команды включает:
1. **IF (Instruction Fetch)** — загрузка команды в CR.
2. **AF (Address Fetch)** — вычисление адреса операнда:
   - Для абсолютной: `адрес = командные биты 0–10`
   - Для относительной: адрес вычисляется по режиму (табл. B.10).
3. **OF (Operand Fetch)** — чтение операнда из памяти в DR.
4. **EX (Execution)** — выполнение операции в АЛУ, запись результата (если требуется), установка флагов.
5. **INT** — проверка прерываний.

#### **4.2. Пример: выполнение `ADD 21`**
1. Команда `0x4021` загружается из памяти в CR.
2. Адрес `21` (0x015) извлекается → AR = 21.
3. Содержимое ячейки 21 → DR.
4. `AC + DR → AC`, флаги N, Z, V, C устанавливаются по результату.

#### **4.3. Пример: выполнение `SWAM ARRAY+` (автоинкремент)**
1. Адрес `ARRAY` вычисляется как `IP + смещение`.
2. Содержимое `[ARRAY]` читается в DR.
3. `AC ↔ DR`, результат записывается обратно в `[ARRAY]`.
4. Адрес в ячейке `ARRAY` **автоматически увеличивается на 1** (для следующей итерации).

---

### **5. Сравнение с безадресными и ветвящимися командами**

| Характеристика | Адресные команды | Безадресные | Ветвления |
|----------------|------------------|-------------|----------|
| Обращение к памяти | Да | Нет (кроме стека) | Только для чтения команды |
| Длина команды | 16 бит | 16 бит | 16 бит |
| Адрес операнда | В команде (11 или 8 бит) | Не требуется | Смещение (8 бит) |
| Примеры | `LD M`, `ADD M`, `ST M` | `CLA`, `HLT`, `PUSH` | `BEQ`, `BNE`, `JUMP` |

---

### **6. Заключение**

Адресные команды БЭВМ-NG обеспечивают **гибкий и эффективный доступ к оперативной памяти**, поддерживая:
- **абсолютную адресацию** для прямого доступа к данным,
- **относительную адресацию** для позиционно-независимого кода,
- **косвенные режимы** для работы с массивами и стеком,
- **непосредственную загрузку** для констант.

Эта система адресации демонстрирует **рациональное использование 16-битного слова**, сочетая простоту реализации с выразительной мощностью, достаточной для написания сложных программ на ассемблере БЭВМ.
