# 25. Безадресные команды

### **1. Определение и формат**

Согласно **разделу 1.3** методических указаний и **рисунку B.2а**, **безадресные команды** — это машинные инструкции, **не содержащие в своём коде адреса операнда или смещения**. Они работают **только с внутренними регистрами процессора** (в первую очередь — с аккумулятором AC и флагами) и **не обращаются к памяти** для загрузки данных.

**Формат безадресной команды** (16 бит):
- **Биты 15–12**: `0000` — код операции (КО = 0).
- **Биты 11–0**: **расширение кода операции** (12 бит), однозначно определяющее конкретную операцию.

> Пример:  
> - `CLA` → `0x0200`  
> - `HLT` → `0x0100`  
> - `ROL` → `0x0400`

---

### **2. Назначение безадресных команд**

Безадресные команды предназначены для выполнения **внутренних операций процессора**, таких как:
- модификация содержимого **аккумулятора (AC)**,
- управление **флагами состояния (N, Z, V, C)**,
- работа со **стеком** (PUSH, POP),
- управление **потоком выполнения** (RET, IRET),
- **остановка процессора** (HLT).

Эти команды **не требуют доступа к оперативной памяти**, что делает их **высокоскоростными** и идеальными для частых вспомогательных операций.

---

### **3. Полный перечень безадресных команд (таблица B.3)**

| Мнемоника | Код | Описание | Изменение флагов |
|----------|-----|---------|------------------|
| `NOP` | `0x0000` | Пустая операция (No Operation) | — |
| `HLT` | `0x0100` | Останов процессора | — |
| `CLA` | `0x0200` | Очистка аккумулятора: `0 → AC` | N=0, Z=1, V=0 |
| `NOT` | `0x0280` | Инверсия AC: `~AC → AC` | N*, Z*, V=0 |
| `CLC` | `0x0300` | Сброс флага переноса: `0 → C` | C=0 |
| `CMC` | `0x0380` | Инверсия флага переноса | C = ~C |
| `ROL` | `0x0400` | Циклический сдвиг влево через флаг C | N*, Z*, V*, C* |
| `ROR` | `0x0480` | Циклический сдвиг вправо через флаг C | N*, Z*, V*, C* |
| `ASL` | `0x0500` | Арифметический сдвиг влево | N*, Z*, V*, C* |
| `ASR` | `0x0580` | Арифметический сдвиг вправо | N*, Z*, V*, C* |
| `SXTB` | `0x0600` | Расширение знака младшего байта AC7 → AC15..AC8 | N*, Z*, V=0 |
| `SWAB` | `0x0680` | Обмен байтами в AC: AC7..0 ↔ AC15..8 | N*, Z*, V=0 |
| `INC` | `0x0700` | Увеличение AC на 1: `AC + 1 → AC` | N*, Z*, V*, C* |
| `DEC` | `0x0740` | Уменьшение AC на 1: `AC - 1 → AC` | N*, Z*, V*, C* |
| `NEG` | `0x0780` | Изменение знака: `~AC + 1 → AC` | N*, Z*, V*, C* |
| `POP` | `0x0800` | Извлечение из стека: `(SP)+ → AC` | N*, Z*, V=0 |
| `POPF` | `0x0900` | Извлечение флагов: `(SP)+ → PS` | N*, Z*, V*, C* |
| `RET` | `0x0A00` | Возврат из подпрограммы: `(SP)+ → IP` | — |
| `IRET` | `0x0B00` | Возврат из прерывания: `(SP)+ → PS, (SP)+ → IP` | N*, Z*, V*, C* |
| `PUSH` | `0x0C00` | Помещение AC в стек: `AC → -(SP)` | — |
| `PUSHF` | `0x0D00` | Помещение PS в стек: `PS → -(SP)` | — |
| `SWAP` | `0x0E00` | Обмен вершины стека с AC | N*, Z*, V=0 |

> **Условные обозначения флагов**:  
> `*` — флаг устанавливается по результату операции,  
> `0` — флаг сбрасывается,  
> `—` — флаг не изменяется.

---

### **4. Особенности выполнения**

#### **4.1. Отсутствие обращения к памяти**
Безадресные команды **не проходят микроциклы AF (Address Fetch) и OF (Operand Fetch)**. Их выполнение ограничивается:
- **IF (Instruction Fetch)** — загрузка команды,
- **EX (Execution)** — выполнение операции в АЛУ или управление стеком/флагами,
- **INT (Interruption)** — проверка прерываний.

Это делает их **самыми короткими по времени выполнения**.

#### **4.2. Работа со стеком**
Команды `PUSH`, `POP`, `CALL`, `RET`, `IRET` используют **указатель стека (SP)**. Хотя они технически обращаются к памяти (стек размещён в ОЗУ), **адрес не задаётся в команде**, поэтому они относятся к безадресным.

#### **4.3. Изменение флагов**
Большинство безадресных команд, работающих с AC (`CLA`, `NOT`, `INC`, `DEC`, `NEG`, `ROL` и др.), устанавливают флаги **N, Z, V, C** по результату, что позволяет использовать их в условных переходах.

Пример:
```asm
CLA     ; AC = 0 → Z = 1
BEQ OK  ; переход выполнится, так как Z = 1
```

---

### **5. Примеры использования**

#### **5.1. Инициализация**
```asm
CLA     ; очистить аккумулятор перед вычислениями
```

#### **5.2. Управление флагами**
```asm
CLC     ; сбросить перенос перед сложением с учётом переноса
ADC M
```

#### **5.3. Работа со стеком**
```asm
PUSH    ; сохранить AC перед вызовом подпрограммы
CALL SUB
POP     ; восстановить AC
```

#### **5.4. Возврат из прерывания**
```asm
IRET    ; восстановить IP и PS из стека, продолжить основную программу
```

#### **5.5. Остановка**
```asm
HLT     ; остановить процессор (для отладки или завершения программы)
```

---

### **6. Отличие от адресных и ветвящихся команд**

| Характеристика | Безадресные | Адресные | Ветвления |
|----------------|------------|----------|----------|
| Обращение к памяти | Нет (кроме стека) | Да | Только чтение команды |
| Длина выполнения | Минимальная | Средняя/длинная | Короткая |
| Адрес в команде | Отсутствует | Присутствует | Смещение (8 бит) |
| Примеры | `CLA`, `HLT`, `PUSH` | `LD M`, `ADD M` | `BEQ`, `JUMP` |

---

### **7. Заключение**

Безадресные команды — **фундаментальный компонент системы команд БЭВМ-NG**, обеспечивающий:
- быструю работу с аккумулятором и флагами,
- эффективное управление стеком и подпрограммами,
- гибкое управление процессом выполнения.

Их простота и скорость делают их незаменимыми в **циклах, процедурах, обработчиках прерываний и инициализационных блоках**.
