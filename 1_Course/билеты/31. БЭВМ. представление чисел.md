### **31. БЭВМ: представление чисел**

В **БЭВМ-NG** все данные представлены **16-битными целыми числами с фиксированной точкой**. Дробные и числа с плавающей точкой **не поддерживаются на уровне команд**.

#### **Типы чисел**
1. **Беззнаковые целые**:  
   - Диапазон: \(0 … 65\,535\) (\(0x0000 … 0xFFFF\)).
2. **Знаковые целые** (в дополнительном коде):  
   - Диапазон: \(-32\,768 … +32\,767\)  
     (\(0x8000 = -32\,768\), \(0xFFFF = -1\), \(0x7FFF = +32\,767\)).

> **Важно**: процессор **не хранит тип числа** — интерпретация зависит от контекста (команд ветвления).

#### **Арифметика и флаги**
Команды (`ADD`, `SUB`, `CMP` и др.) выполняют одинаковые битовые операции, но результат интерпретируется через флаги:

| Флаг | Назначение | Использование |
|------|-----------|----------------|
| **C** | Перенос из старшего бита | Беззнаковая арифметика (`BCC`, `BCS`) |
| **V** | Знаковое переполнение | Знаковая арифметика (`BLT`, `BGE`) |
| **N** | Знак результата | Знаковая арифметика |
| **Z** | Результат = 0 | Обе арифметики |

#### **Пример**
Число \(-709\):  
- \(+709 = 0x02C5\)  
- Инверсия: \(0xFD3A\)  
- +1 → \(0xFD3B\)  
Проверка: \(0x02C5 + 0xFD3B = 0x10000 \rightarrow 0\) (перенос игнорируется).

#### **Ограничения**
- Нет аппаратной поддержки дробных чисел.  
- Для дробей — программное масштабирование (например, ×1000).
# 31. БЭВМ: представление чисел

### **1. Общая характеристика**

В **БЭВМ-NG** все числовые данные представлены в **16-битных машинных словах** и интерпретируются **исключительно как целые числа с фиксированной точкой**.  
Дробные числа и числа с плавающей точкой **не поддерживаются на уровне машинных команд**.

> 💡 Это упрощает архитектуру и делает модель идеальной для изучения основ двоичной арифметики.

---

### **2. Типы чисел в БЭВМ-NG**

Согласно **разделу 1.4.1** методических указаний, 16-битное слово может интерпретироваться двумя способами:

#### **2.1. Беззнаковые целые числа**
- Все 16 бит используются для представления величины.
- **Диапазон**:  
  $$
  0 \leq N \leq 2^{16} - 1 = 65\,535
  $$
- **Примеры**:  
  - `0x0000` = 0  
  - `0xFFFF` = 65 535

#### **2.2. Знаковые целые числа**
- Используется **дополнительный код (two’s complement)**.
- Старший (15-й) бит — **знаковый**:  
  - `0` → положительное число,  
  - `1` → отрицательное число.
- **Диапазон**:  
  $$
  -2^{15} \leq N \leq 2^{15} - 1 \quad \Rightarrow \quad -32\,768 \leq N \leq +32\,767
  $$
- **Примеры**:  
  - `0x0000` = 0  
  - `0x7FFF` = +32 767  
  - `0x8000` = –32 768  
  - `0xFFFF` = –1

> ⚠️ **Важно**: процессор **не хранит информацию о типе числа**. Одно и то же слово (`0xFFFF`) может быть как 65 535 (беззнаковое), так и –1 (знаковое) — интерпретация зависит от используемых команд ветвления.

---

### **3. Арифметические операции**

Команды `ADD`, `SUB`, `INC`, `DEC`, `CMP` и др. выполняют **одни и те же битовые операции** независимо от типа числа.  
Однако **результат интерпретируется по-разному** с помощью флагов:

| Флаг | Назначение | Используется для |
|------|-----------|------------------|
| **C (Carry)** | Перенос из старшего бита | Беззнаковая арифметика |
| **V (Overflow)** | Переполнение знакового результата | Знаковая арифметика |
| **N (Negative)** | Знак результата (старший бит) | Знаковая арифметика |
| **Z (Zero)** | Результат = 0 | Обе арифметики |

> 💡 Пример:  
> После `ADD`:
> - Если использовать `BCC` → сравнение **беззнаковое**,
> - Если использовать `BLT` → сравнение **знаковое**.

---

### **4. Пример: представление числа –709**

В методических указаниях (раздел 1.4.1) приведён пошаговый пример:

1. Модуль числа:  
   \( 709_{10} = 0000\ 0010\ 1100\ 0101_2 = 0x02C5 \)
2. Инверсия битов:  
   \( \sim 0x02C5 = 1111\ 1101\ 0011\ 1010_2 = 0xFD3A \)
3. Прибавление 1:  
   \( 0xFD3A + 1 = 0xFD3B \)

Проверка:  
\( 0x02C5 + 0xFD3B = 0x10000 \) → перенос игнорируется → результат = 0 ✅  
Таким образом, **–709 = 0xFD3B** в дополнительном коде.

---

### **5. Отсутствие поддержки дробных чисел**

Прямо указано в методичке:

> «БЭВМ **не умеет на уровне машинных команд работать с числами с плавающей точкой**».

Следовательно:
- Все вычисления — **целочисленные**,
- Для эмуляции дробей программист должен использовать **масштабирование** (например, хранить значения в «копейках» или умножать на 1000).

---

### **6. Заключение**

Представление чисел в БЭВМ-NG — это **упрощённая, но полная модель целочисленной арифметики**, охватывающая:
- беззнаковые и знаковые целые,
- дополнительный код,
- работу с флагами C и V.

Эта модель **отражает принципы реальных процессоров**, но исключает сложность плавающей точки, чтобы сосредоточиться на **фундаментальных понятиях**: разрядность, перенос, переполнение, дополнительный код и флаги состояния.
