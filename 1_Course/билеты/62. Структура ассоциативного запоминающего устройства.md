### **62. Структура ассоциативного запоминающего устройства (АЗУ / CAM)**

**Ассоциативная память (CAM — Content-Addressable Memory)** — устройство, в котором данные ищутся **по содержимому**, а не по адресу.

#### **Отличие от RAM**
| Характеристика | RAM | CAM |
|----------------|-----|-----|
| Доступ | По адресу | По содержимому |
| Поиск | O(N) | O(1) (параллельный) |
| Стоимость | Низкая | Высокая |
| Энергопотребление | Умеренное | Высокое |

#### **Типы**
1. **Fully Associative CAM** — поиск по всем ячейкам одновременно.
2. **TCAM (Ternary CAM)** — поддержка маски `X` («don’t care»), используется в сетях для маршрутизации.

#### **Структура ячейки**
- **Binary CAM**: SRAM + компаратор (XNOR + AND).
- **TCAM**: две SRAM-ячейки на бит — значение и маска.

#### **Принцип поиска**
1. Искомое слово подаётся на все ячейки.
2. Каждая ячейка сравнивает своё содержимое с запросом.
3. Совпадение → выставляется флаг.
4. Контроллер возвращает адрес совпавшей ячейки за 1 такт.
#### **Применение**
- **Сетевые коммутаторы/маршрутизаторы** (TCAM для таблиц маршрутизации),
- **TLB** (Translation Lookaside Buffer),
- **Branch Target Buffer**,
- **Фильтрация пакетов**.
#### **Недостатки**
- Высокая стоимость и энергопотребление,
- Ограниченный объём (обычно КБ–МБ).
#### **В БЭВМ-NG**
- Не рассматривается: модель использует **адресуемую память** с прямым доступом по адресу.

# 62. Структура ассоциативного запоминающего устройства

### **1. Определение**

**Ассоциативное запоминающее устройство (АЗУ)**, также известное как **CAM (Content-Addressable Memory)** — это специализированный тип памяти, в котором **поиск данных осуществляется не по адресу, а по их содержимому**.

В отличие от традиционной RAM (где: «дай данные по адресу 0x123»), в CAM:  
> «Есть ли ячейка, содержащая значение `0xABCD`? Если да — верни её адрес».

---

### **2. Основное отличие от адресуемой памяти**

| Характеристика | RAM (адресуемая) | CAM (ассоциативная) |
|----------------|------------------|---------------------|
| **Принцип доступа** | По адресу | По содержимому |
| **Скорость поиска** | O(1) для доступа, O(N) для поиска | **O(1)** для полного параллельного поиска |
| **Аппаратная сложность** | Низкая | Очень высокая |
| **Энергопотребление** | Умеренное | Высокое (все ячейки работают одновременно) |
| **Стоимость** | Низкая | Высокая |

---

### **3. Типы ассоциативной памяти**

#### **3.1. Полностью ассоциативная (Fully Associative CAM)**
- Любое слово может быть сохранено в **любой ячейке**.
- При поиске **все ячейки сравниваются параллельно** с искомым значением.
- **Максимальная гибкость**, но **максимальная сложность**.

#### **3.2. Частично ассоциативная (Content-Addressable, но с маской)**
- Поддерживает **маскирование битов** (например, «игнорировать последние 8 бит»).
- Используется в **TCAM (Ternary CAM)**.

#### **3.3. TCAM (Ternary CAM)**
- Каждый бит может иметь **три состояния**: `0`, `1`, `X` («don’t care»).
- Позволяет выполнять **поиск по шаблону**.
- **Ключевое применение**: **таблицы маршрутизации** в сетевых маршрутизаторах.
  - Пример: найти маршрут для `192.168.0.0/16` → маска `11111111 11111111 00000000 00000000`.

---

### **4. Структура ячейки CAM**

#### **4.1. Двоичная CAM-ячейка (Binary CAM)**
Состоит из:
- **Ячейки памяти** (обычно SRAM-триггер: 2 инвертора + 2 транзистора),
- **Схемы сравнения** (XNOR + AND), которая сравнивает хранимый бит с входным.

Для одного бита:
- Если `stored_bit == input_bit` → выход сравнения = 1,
- Иначе = 0.

Для слова из N бит: выходы всех N компараторов объединяются через AND — **совпадение только если все биты совпадают**.

#### **4.2. Ternary CAM-ячейка (TCAM)**
Содержит **две SRAM-ячейки на бит**:
- Одна хранит значение (`0` или `1`),
- Вторая — маску (`1` = учитывать, `0` = «don’t care»).

Это позволяет реализовать третий логический уровень (`X`).

---

### **5. Схема поиска**

1. Искомое слово (или шаблон) подаётся **на все ячейки одновременно**.
2. Каждая ячейка **параллельно сравнивает** своё содержимое с запросом.
3. Если совпадение — ячейка выставляет **флаг совпадения (match line)**.
4. Контроллер определяет:
   - Есть ли совпадение?
   - Если да — **возвращает адрес(а) совпавших ячеек**.

> 💡 Поиск занимает **ровно один такт**, независимо от объёма памяти.

---

### **6. Применение CAM**

| Область | Использование |
|--------|--------------|
| **Сети** | - Таблицы MAC-адресов в коммутаторах<br>- Таблицы маршрутизации (TCAM)<br>- Фильтрация пакетов (ACL) |
| **Кэш-память** | Частично ассоциативный кэш использует **ограниченную ассоциативность** (но не полную CAM-архитектуру из-за стоимости) |
| **Процессоры** | - TLB (Translation Lookaside Buffer)<br>- Branch Target Buffer |
| **ИИ и обработка данных** | Поиск образцов в реальном времени |

> 🔍 **Пример**: сетевой коммутатор получает кадр с MAC-адресом получателя `aa:bb:cc:dd:ee:ff`.  
> TCAM за **1 такт** находит порт, связанный с этим адресом, и пересылает кадр — без сканирования таблицы.

---

### **7. Недостатки CAM**

- **Высокая стоимость**: каждая ячейка содержит **и память, и логику сравнения** → в 10–20 раз дороже SRAM.
- **Высокое энергопотребление**: при каждом поиске задействованы **все ячейки**.
- **Ограниченный объём**: CAM-чипы обычно имеют **десятки–сотни килобайт**, редко — мегабайты.

---

### **8. CAM и БЭВМ-NG**

В методических указаниях по БЭВМ-NG **ассоциативная память не рассматривается**. БЭВМ использует:
- **Классическую адресуемую память**,
- **Плоское адресное пространство**,
- **Прямой доступ по 11-битному адресу**.

CAM — это **специализированная аппаратура**, не входящая в базовую модель учебной ЭВМ.

---

### **9. Заключение**

Ассоциативное запоминающее устройство — это **высокопроизводительный, но дорогостоящий** компонент, решающий задачу **мгновенного поиска по содержимому**. Несмотря на свою узкую специализацию, CAM **критически важен** в сетевом оборудовании, кэшировании и системах реального времени, где скорость поиска важнее стоимости и энергопотребления.

> 📌 **Примечание**: Тема **не освещается в методических указаниях по БЭВМ-NG**, так как модель фокусируется на **адресуемой памяти**.
