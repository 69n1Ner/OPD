[[9. Интерпретатор команд (shell)]]

### **10.1. Основные операторы**

|   |   |
|---|---|
|`>`|Перезапись stdout|
|`>>`|Добавление в stdout|
|`<`|Ввод из файла (stdin)|
|`2>`|Перезапись stderr|
|`2>>`|Добавление в stderr|
|`&>`|stdout + stderr в файл|
|`2>&1`|stderr → stdout|
|`1>&2`|stdout → stderr|

### **10.2. Примеры**

```
# Только данные в файл, ошибки — в терминал

grep "error" /var/log/* > errors.txt

# Ошибки в отдельный файл

gcc prog.c 2> compile_errors.log

# Всё в один файл

make &> build.log 

# stderr в stdout → в next_cmd

ls /nonexist 2>&1 | grep "No such"  

# Ввод из файла

sort < data.txt
```
### **10.3. Закрытие потока**
```
command 2>&- # закрыть stderr
```

### **10.4. Приоритет и порядок**

Порядок важен:

```
# ОШИБКА: stderr не перенаправлен

cmd 2>&1 > out.txt
# ПРАВИЛЬНО: сначала stdout в файл, затем stderr туда же

cmd > out.txt 2>&1
```

> 📌 **В БЭВМ-NG**:  
> Перенаправления **как таковых нет**, но **ввод-вывод программируется явно**:
> 
> - `IN #4` — чтение из ВУ-2 (аналог stdin),
> - `OUT #2` — запись в ВУ-1 (аналог stdout).  
>     Нет механизма **динамического перенаправления** на этапе запуска — всё задаётся в коде программы.  
>     Например, в листинге B.14 (стр. 55):
> 
> asm
