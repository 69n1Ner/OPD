# 11. Фильтры

### **1. Определение фильтра в контексте UNIX/Linux**

**Фильтр** — это команда (программа), которая:
- читает данные из **стандартного ввода** (`stdin`, файловый дескриптор 0),
- обрабатывает их (преобразует, выбирает, сортирует и т.д.),
- выводит результат в **стандартный вывод** (`stdout`, файловый дескриптор 1).

Фильтры **не изменяют исходные файлы**, работают **потоково** (строка за строкой) и **не требуют интерактивного ввода**. Это делает их идеальными для **композиции через конвейеры** (`|`).

> ⚠️ **Примечание**: В методических указаниях по **БЭВМ-NG** термин «фильтр» **не используется** и **не определяется**, поскольку модель не включает операционную систему, файловую систему или командную оболочку. Поэтому всё содержание этого ответа основано на **общепринятой UNIX-терминологии**.

---

### **2. Основные характеристики фильтров**

1. **Потоковая обработка**: данные обрабатываются по мере поступления, без необходимости загружать весь файл в память.
2. **Независимость от источника/приёмника**: фильтр не «знает», откуда пришли данные (файл, клавиатура, другой процесс) и куда они уйдут.
3. **Модульность и композиция**: один фильтр выполняет одну задачу, но может быть объединён с другими.

---

### **3. Примеры стандартных фильтров в UNIX/Linux**

| Команда | Назначение | Пример использования |
|--------|-----------|----------------------|
| `cat` | Вывод содержимого файла(ов) | `cat file.txt` |
| `grep` | Поиск строк по шаблону | `grep "error" log.txt` |
| `sort` | Сортировка строк | `sort names.txt` |
| `uniq` | Удаление дублирующихся строк (только подряд идущих) | `sort file \| uniq` |
| `cut` | Извлечение полей (столбцов) | `cut -d: -f1 /etc/passwd` |
| `head` | Вывод первых N строк | `head -10 log.txt` |
| `tail` | Вывод последних N строк | `tail -f /var/log/syslog` |
| `wc` | Подсчёт строк, слов, байтов | `wc -l *.c` |
| `tr` | Замена/удаление символов | `echo "HELLO" \| tr 'A-Z' 'a-z'` |
| `sed` | Потоковый редактор текста | `sed 's/foo/bar/g' file` |
| `awk` | Язык обработки текстовых данных | `awk '{print $1}' data.txt` |

---

### **4. Использование фильтров в конвейерах (pipes)**

Конвейер (`|`) — ключевой механизм UNIX для связывания фильтров:

```bash
# Пример: найти все процессы пользователя, отсортировать и удалить дубликаты
ps aux | grep $USER | awk '{print $11}' | sort | uniq
```

Этот пример демонстрирует **философию UNIX**: «делай одну вещь и делай её хорошо».

---

### **5. Фильтры и перенаправления**

Фильтры могут сочетаться с перенаправлениями:

```bash
# Вывод фильтра записать в файл
ls -l | grep ".txt" > text_files.txt

# Прочитать из файла, обработать, записать в другой
cat input.txt | tr 'a-z' 'A-Z' | sort > output.txt
```

---

### **6. Создание собственного фильтра**

Любой скрипт или программа, читающая из `stdin` и пишущая в `stdout`, является фильтром. Пример на Bash:

```bash
#!/bin/bash
# myfilter.sh — простой фильтр
while IFS= read -r line; do
  echo "[$(date '+%H:%M')] $line"
done
```

Использование:
```bash
echo "Hello" | ./myfilter.sh
# Вывод: [14:30] Hello
```

---

### **7. Отличие от не-фильтров**

Не все команды являются фильтрами:
- `cp`, `mv`, `rm` — работают с файлами напрямую, **не через stdin/stdout**.
- `ls` — может выводить в stdout, но **не читает stdin**, поэтому сам по себе не фильтр (хотя его вывод можно подать в фильтр).

---

### **8. Заключение**

Фильтры — фундаментальный инструмент UNIX-систем, обеспечивающий:
- гибкость обработки текстовых данных,
- повторное использование программ,
- простоту отладки и тестирования.

Их эффективность раскрывается именно в **комбинации** с другими фильтрами через конвейеры.

> 📌 **Важно**: В БЭВМ-NG **нет фильтров**, так как отсутствует ОС, файловая система и понятие потоков. Ввод-вывод в БЭВМ реализуется **явными командами `IN` и `OUT`**, работающими с регистрами устройств (см. раздел 2 методички).
